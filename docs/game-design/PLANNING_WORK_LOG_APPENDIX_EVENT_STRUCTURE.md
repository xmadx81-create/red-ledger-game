# 본격 기획 단계 작업 로그 부록 - 이벤트 구조

## 작업 주제

이벤트·선택지 구조 작성

## 진행 내용

1. `docs/game-design/EVENT_STRUCTURE.md`를 생성했다.
2. 이벤트 발생, 상황 설명, 인물 반응, 선택지, 자원 변화, 관계 변화, 단서 갱신, 후속 이벤트로 이어지는 기본 흐름을 정의했다.
3. 이벤트 분류를 운영, 가문, 인간 사회, 언론, 내부 조직, 암거래, 캐릭터, 위기 이벤트로 나누었다.
4. 이벤트 ID와 선택지 ID 규칙을 정의했다.
5. 선택지 톤을 정석, 은폐, 가문 우선, 인간 우선, 중용, 위험 거래, 지연으로 나누었다.
6. Day 1~Day 7 핵심 이벤트 초안을 정리했다.
7. 인물 반응, 관계 상태, 엔딩 플래그, Google Sheets 연결 기준을 정리했다.

## 결정 사항

- 이벤트는 단순 퀘스트가 아니라 행정 누아르식 위기로 설계한다.
- 선택지는 선악보다 운영 철학과 대가를 기준으로 나눈다.
- 모든 선택지는 자원 변화와 최소 하나 이상의 후속 효과를 가져야 한다.
- Day 7에서는 누적 결과를 평가해 엔딩 플래그를 판정한다.
