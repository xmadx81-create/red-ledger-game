# 히든 합성 트리 001 (비공개 최단 전설 루트)

작성일: 2026-06-16
상위 문서: `RACE_HYBRID_FUSION_SYSTEM_001.md`, `LEGENDARY_TOPDOWN_FUSION_STRUCTURE_001.md`
데이터: `hidden-fusion-recipes.json`
문서 성격: 일반 합성의 예외 — 비공개 레시피 체인 정형화 (카드 미정, 구조만)

> 아이디어: 일반 확률 합성과 별개로, **특정 카드 조합을 정확히 맞추면 50%라는 높은 확률로 등업**되고, 이를 연쇄하면 **전설까지 가는 비공개 최단 루트**가 된다. 정답 조합은 숨겨져 있어 발견·공략의 재미를 만든다.

---

## 1. 왜 만드는가

```text
일반 합성: 등급별 낮은 확률 + 천장 → 전설까지 매우 오래 (N 환산 수만 장).
히든 루트: '비밀 조합'을 알면 50% 등업을 연쇄해 훨씬 빠르게 전설 도달.
→ 정답을 찾는 발견의 재미, 공략/커뮤니티 메타, "안다 vs 모른다"의 격차.
```

이는 "합성에 미치게" 하는 동기를 강화한다. 무지성 양산이 아니라 **정답 탐색**이 보상받는다.

---

## 2. 핵심 규칙

```text
- 매칭 기준: 특정 카드 '조합'(정확한 2장). 등급만 맞으면 되는 일반 합성과 다르다.
- 성공률: 기본 50% (일반 N→R 25% / EP→L 3%보다 훨씬 높음).
- 성공: 지정된 상위 결과 카드 1장 생성(새 cardUid). 입력 소비.
- 실패: 입력 소비(또는 일부 환원 — 추후). 천장 정책은 일반과 별도/미적용.
- 비공개: UI에 노출하지 않음. 힌트/로어/이벤트/커뮤니티로만 발견.
- 연쇄: 결과 카드 + 또 다른 특정 카드 → 다음 등급. N→R→SR→EP→L까지 사슬.
```

---

## 3. 체인 구조 (예시 스켈레톤, 카드 미정)

`hidden-fusion-recipes.json`의 `HIDDEN-CHAIN-001` (전부 placeholder `TBD-*`):

| 단계 | 입력(2장) | 결과 | 확률 |
|---:|---|---|---:|
| 1 | TBD-N-α1 + TBD-N-α2 | TBD-R-α | 50% |
| 2 | TBD-R-α + TBD-?-α3 | TBD-SR-α | 50% |
| 3 | TBD-SR-α + TBD-?-α4 | TBD-EP-α | 50% |
| 4 | TBD-EP-α + TBD-?-α5 | TBD-L-α (전설) | 50% |

→ 4번의 50% 등업으로 전설 도달(평균 ~8회 시도 + 특정 카드 확보). 일반 루트 대비 압도적 단축.

---

## 4. 균형/악용 방지

```text
- 확정이 아니라 50% RNG → 양산 억제.
- 입력에 희귀/특정 카드를 끼워 무한 반복 불가하게 설계 가능.
- 단계가 오를수록 입력 난도를 올려, 최단 루트라도 비용이 들게 함.
- 전설별로 서로 다른 비공개 체인을 둬 다양성 확보.
```

---

## 5. 일반 합성과의 관계

```text
fusion-recipe-tree.json : 등급별 fungible 확률 합성(공개, 기본 루트).
hidden-fusion-recipes.json : 특정 카드 조합 기반 비공개 예외(고확률 최단 루트).
두 시스템은 공존한다. 모르면 일반 루트로, 알면 히든 루트로.
```
