# 이벤트·선택지 구조

이 문서는 `red-ledger-game`의 이벤트 발생, 선택지, 결과 처리, 자원 변화, 인물 반응, 후속 분기 구조를 정의합니다.

## 목적

이벤트 시스템은 플레이어가 매일 마주하는 사건과 판단의 기본 구조입니다.

`적혈의 장부`의 이벤트는 단순한 퀘스트가 아니라, 혈연센터의 장부, 보고, 민원, 가문 명령, 암거래 단서, 인물 관계가 얽혀 발생하는 행정 누아르식 위기입니다.

## 핵심 흐름

```text
이벤트 발생
→ 상황 설명
→ 관련 인물 반응
→ 선택지 제시
→ 자원 변화
→ 관계 변화
→ 단서 갱신
→ 후속 이벤트 또는 일일 보고 반영
```

## 이벤트 분류

| 분류 | 설명 | 예시 |
|---|---|---|
| 운영 이벤트 | 센터의 일상 업무와 자원 관리 사건 | 재고 부족, 폐기 기록, 일정 조정 |
| 가문 이벤트 | 카르테인 가문의 명령과 압박 | 추가 공급 요구, 원로 보고, 가문 징계 |
| 인간 사회 이벤트 | 민원, 후원자, 외부 감사, 시민 반응 | 민원 접수, 캠페인 요청, 감사 방문 |
| 언론 이벤트 | 기자, 제보, 보도 위험 관련 사건 | 정하린 기자의 취재, 익명 제보 |
| 내부 조직 이벤트 | 직원 불안, 실수, 배신, 피로 누적 | 야간 근무 거부, 기록 누락 |
| 암거래 이벤트 | 인간들끼리 혈액을 거래하는 비공식 흐름 | 사라진 혈액, 뒷문 거래, 수상한 주문 |
| 캐릭터 이벤트 | 주요 인물의 개인 서사와 관계 변화 | 김도현의 고백, 박세연의 비밀 접촉 |
| 위기 이벤트 | 특정 수치가 위험 구간에 들어섰을 때 발생 | 폭주 징후, 언론 폭로 직전, 내부 붕괴 |

## 선택지 톤 분류

| 톤 | 의미 |
|---|---|
| 정석 | 표면 조직의 규칙과 절차를 따름 |
| 은폐 | 기록 조정, 회유, 비공식 처리 사용 |
| 가문 우선 | 카르테인 가문의 요구를 우선함 |
| 인간 우선 | 인간 사회의 신뢰와 안전을 우선함 |
| 중용 | 양쪽 모두를 완전히 만족시키지 않지만 균형을 추구함 |
| 위험 거래 | 암거래망, 뒷문 계약, 고위험 수단 사용 |
| 지연 | 당장 결정을 미루고 후속 위험을 감수함 |

## 결과 처리 원칙

모든 선택지는 최소 하나 이상의 결과를 가져야 합니다.

결과는 다음 범주 중 일부를 포함합니다.

- 자원 변화
- 인물 반응
- 관계 상태 변화
- 암거래 단서 변화
- 후속 이벤트 개방
- 실패 조건 누적
- 엔딩 플래그 갱신

## 자원 변화 표기

내부 데이터에는 숫자 변화를 기록합니다.

예시:

```text
혈액 재고 +8
혈액 수요 -3
언론 노출 +4
보안 등급 -6
가문 만족도 +5
조직 불안 +3
암거래 단서 +10
```

플레이어 UI에서는 숫자를 모두 공개하지 않고 다음처럼 표현할 수 있습니다.

```text
혈액 재고 증가
언론 노출 소폭 증가
보안 등급 하락
암거래 단서 크게 증가
```

## 이벤트 발생 조건

| 조건 | 설명 |
|---|---|
| 고정 Day | 특정 Day에 반드시 발생 |
| 자원 조건 | 특정 자원이 일정 수치 이상/이하일 때 발생 |
| 선택 결과 | 이전 선택에 따라 발생 |
| 인물 관계 | 특정 캐릭터 관계 상태에 따라 발생 |
| 암거래 단서 | 암거래 단서 수치에 따라 발생 |
| 실패 누적 | 공급 부족, 은폐 실패 등이 누적될 때 발생 |
| 랜덤 가중치 | MVP 이후 확장용 |

| Day 1 | 정석 운영 / 가문 우선 / 기록 보정 |

| Day 3 | 공개 설명 / 기자 회유 / 기록 정정 |
| Day 4 | 전면 조사 / 조용한 회수 / 보고서상 오차 처리 |
| Day 5 | 요구 전부 수용 / 일부 협상 / 대체 경로 탐색 |
| Day 6 | 암거래망 회수 / 암거래망 이용 / 내부자 색출 / 기자 활용 |
| Day 7 | 최종 보고 / 책임 전가 / 가문 우선 보고 / 인간 사회 우선 보고 |

## 인물 반응 구조

이벤트 결과는 주요 인물의 관계 상태를 바꿀 수 있습니다.

| 인물 | 반응 방향 |
|---|---|
| 서윤재 | 중용 실패 시 내적 갈등 또는 결단 변화 |
| 김도현 | 야간 처리 남용 시 불안 증가, 보호받으면 충성 상승 |
| 박세연 | 위험 거래 선택 시 협력 상승 또는 배신 위험 증가 |
| 정하린 | 은폐를 반복하면 적대 상승, 암거래 정보를 공유하면 조건부 협력 가능 |
| 엘리엇 카르테인 | 수요 충족 시 만족 상승, 지연 또는 실패 시 압박 증가 |

## 관계 상태

| 상태 | 의미 |
|---|---|
| 신뢰 | 추가 선택지 또는 완화 효과 제공 |
| 협력 | 조건부 도움 제공 |
| 의심 | 감시와 압박 증가 |
| 압박 | 요구 강제 또는 자원 소모 증가 |
| 적대 | 사건 악화 또는 위기 유발 |
| 잠복 | 겉으로는 협력하지만 배신 가능성 존재 |

## 엔딩 플래그

선택 결과는 엔딩 플래그를 누적할 수 있습니다.

| 플래그 | 의미 |
|---|---|
| SUPPLY_STABLE | 수요/공급 안정 |
| SUPPLY_COLLAPSE | 공급 붕괴 위험 누적 |
| EXPOSED_MEDIA | 언론 노출 누적 |
| FAMILY_DOMINANT | 가문 우선 선택 누적 |
| HUMAN_TRUST | 인간 사회 우선 선택 누적 |
| BLACK_MARKET_DOMINANT | 암거래망 방치 또는 이용 누적 |
| INTERNAL_COLLAPSE | 조직 불안 누적 |

## MVP 구현 최소 조건

- Day별 핵심 이벤트 1개 이상
- 이벤트별 선택지 3개 이상
- 선택지별 자원 변화 기록
- 인물 반응 1개 이상 연결
- 암거래 단서 변화 연결
- 엔딩 플래그 최소 1개 이상 연결
- Day 7에서 누적 결과 평가
