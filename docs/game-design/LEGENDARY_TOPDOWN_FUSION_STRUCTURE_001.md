# 전설 기준 하향식 합성 구조 001

작성일: 2026-06-15
상위 문서: `CARD_GAME_DESIGN_001.md`, `RACE_HYBRID_FUSION_SYSTEM_001.md`
데이터: `legendary-anchor-set.json`, `fusion-recipe-tree.json`
문서 성격: 전설 우선 정의 → 하위 등급 소요량 역산(top-down) 정형화

> 방법론: **전설을 먼저 데이터로 확정**하고(이미지 제외), 그 정점을 기준으로 **하위 등급이 몇 장 필요한지를 위에서 아래로 역산**해 합성 경제를 세운다. 확정 가챠가 아니라 **확률+천장**이며, 매우 희귀하지만 **노력하면 반드시 도달 가능**하게 한다.

---

## 1. 설계 순서 (하향식)

```text
1. 전설(L) 5장을 데이터로 먼저 확정 (특이성 = 세계관 기둥)   ← 완료
2. 전설 1장 도약에 필요한 EP 소요량 정의
3. EP 1장에 필요한 SR 소요량 정의
4. SR ← R, R ← N 으로 계속 역산
5. 순수 N 환산 누적량 산출 → "전설은 얼마나 멀리 있는가" 확정
6. 중간 등급 직접 공급(드랍)으로 곱셈 폭증 완화
7. (마지막) 일러스트는 Batch 단위·톤 가이드로 생성
```

## 3. 하향식 합성 트리 (역산)

데이터 원본: `fusion-recipe-tree.json` (수치는 v0 초안, 전부 튜닝 대상)

### 3.1 등급 도약 파라미터
같은 등급 재료를 합성해 다음 등급으로 **도약 시도**. 실패하면 베이스는 보존, 재료 소모, 천장 상승. `pityCap`회 실패 후 확정 성공.

| 도약 | 성공률 | 천장(pity) | 기대 시도 | 출력 1장당 소요(기대→최악) |
|---|---:|---:|---:|---|
| N→R | 25% | 8 | 4.0 | 5 → 9 (N) |
| R→SR | 15% | 12 | 6.7 | 7.7 → 13 (R) |
| SR→EP | 8% | 20 | 12.5 | 13.5 → 21 (SR) |
| EP→L | 3% | 40 | 33.3 | 34.3 → 41 (EP) |

> EP(영웅)는 흔히 말하는 **SSR 위치**다. "전설이 되려면 SSR(=EP)이 몇 장 필요한가"가 바로 EP→L 행이다.

### 3.2 순수 N 환산 (전설은 얼마나 먼가)
중간 등급을 전혀 직접 공급받지 않고 N부터만 쌓을 때:

| 목표 | 기대 N 환산 | 최악(천장) N 환산 |
|---|---:|---:|
| R 1장 | 5 | 9 |
| SR 1장 | 38 | 117 |
| EP 1장 | 518 | 2,457 |
| **L 1장** | **약 17,757** | **약 100,737** |

→ 이 세팅이면 전설 1장은 **순수 N 약 1.7만 장(기대) ~ 10만 장(천장 최악)** 규모의 장기 누적. 곱셈 구조라 **의도적으로 매우 희귀**하지만, **천장으로 도달은 수학적으로 보장**된다.

### 3.3 곱셈 폭증 완화 (필수)
순수 바닥쌓기는 폭증한다. 따라서 **중간 등급도 직접 공급**한다.

```text
N    : 출석/로그인/드랍으로 무료 대량 공급 (합성 연료)
R/SR : 스테이지·던전 드랍으로 직접 공급
EP   : 고난도/주간 콘텐츠 제한 공급 + 합성
L    : 직접 뽑기 없음 — 합성 도약 또는 스토리/시즌 한정 보상
```

직접 공급이 들어오면 실제 전설 도달 부담은 위 환산값보다 크게 줄어든다.

## 4. "확정 가챠 아님" 원칙

```text
- 고정 레시피(예: EP 5장 = L 1장 확정)가 아니다.
- 매 합성은 확률 판정이며 대부분 실패한다.
- 성공률 = 기본확률 + 재료보정 + 행운보정 + 누적보정(천장).
- 천장(pityCap)으로 "꾸준히 하면 반드시 도약"을 보장한다 → 노력 도달 가능.
- 확률·천장 수치는 투명 공개(확률 조작 금지). 실패도 소량 부산물 지급.
```

이는 `CORE_LOOP_REDESIGN_001.md`의 "유저를 괴롭히지 않는다" 윤리 가드레일과 일치한다.
