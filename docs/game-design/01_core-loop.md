# 적혈의 장부 — 핵심 루프

작성일: 2026-06-16

## 1. 1차 MVP 루프

MVP는 `7일 운영`을 기준으로 한다.

```text
하루 시작
→ 현재 자원 확인
→ 일일 이벤트 발생
→ 선택지 결정
→ 자원 변화 적용
→ 인물 관계/위험 변화 적용
→ 일일 결산
→ 다음 날 진행
→ 7일차 최종 결산
→ 엔딩 분기
```

## 2. 플레이어의 역할

플레이어는 혈연센터의 실무 총괄자 역할을 맡는다.

플레이어는 매일 다음을 판단한다.

- 표면 조직의 신뢰를 유지할 것인가
- 이면 조직의 요구를 우선할 것인가
- 보안과 기록을 강화할 것인가
- 자금과 조직 피로를 감수할 것인가
- 언론 노출을 줄일 것인가

## 3. 하루 단위 구조

### 1단계: 상태 확인

표시 자원:

- 혈액 재고
- 가문 자금
- 가문 위신
- 인간 신뢰
- 언론 노출
- 조직 불안
- 보안 등급
- 가문 만족도

### 2단계: 이벤트 발생

이벤트는 조건에 따라 발생한다.

예시:

- 보안 등급이 낮으면 기록 누락 이벤트 발생
- 언론 노출이 높으면 기자 추적 이벤트 발생
- 조직 불안이 높으면 내부 스파이 의심 이벤트 발생

### 3단계: 선택

각 이벤트는 최소 2개의 선택지를 가진다.

선택지는 단순한 정답/오답이 아니라 자원 간 교환 구조를 가진다.

예시:

```text
홍보자료 배포
→ 인간 신뢰 증가
→ 자금 감소
→ 언론 노출 감소
```

```text
제보자 추적
→ 보안 증가
→ 조직 불안 증가
→ 인간 신뢰 감소
```

### 4단계: 결과 적용

선택 결과는 자원, 인물 관계, 위험도에 반영된다.

### 5단계: 결산

하루가 끝나면 다음을 보여준다.

- 증가한 자원
- 감소한 자원
- 위험 신호
- 다음 날 예고

## 4. 7일차 엔딩 분기

7일차 종료 시 아래 지표를 기준으로 엔딩을 결정한다.

- 혈액 재고
- 인간 신뢰
- 언론 노출
- 가문 만족도
- 가문 위신
- 조직 불안

## 5. 설계 원칙

- 매 선택은 최소 2개 이상의 자원에 영향을 준다.
- 한쪽만 계속 선택하면 다른 축이 무너진다.
- 단기 이익과 장기 리스크를 동시에 보여준다.
- 플레이어가 숫자를 통해 조직 운영의 압박을 느끼게 한다.
