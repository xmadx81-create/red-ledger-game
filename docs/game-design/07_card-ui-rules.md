# 적혈의 장부 — 후보E 카드형 UI 규칙

작성일: 2026-06-16

## 1. 목적

이 문서는 `적혈의 장부 / Red Ledger`의 카드형 UI 기준을 정리한다.

현재 기준안은 사용자가 승인한 `후보E` 방향이다.

핵심 방향:

- 카드 형식 통일
- 등급별 테두리 색상 차등
- 일반 등급은 무채색
- 전설 등급은 황금색
- 등급이 올라갈수록 아우라가 강하고 알록달록해짐
- 추후 카드 도감 시스템으로 확장 가능해야 함

## 2. 등급 기준

| 등급 | 색상 방향 | 아우라 방향 | 용도 |
|---|---|---|---|
| 일반 | 무채색 / 회색 | 거의 없음 | 기본 카드 |
| 고급 | 푸른색 | 약한 단색 아우라 | 기능성 인물/아이템 |
| 희귀 | 보라 + 청색 | 다색 아우라 | 중요 인물/사건 카드 |
| 전설 | 황금색 | 강한 황금 + 다색 아우라 | 핵심 인물/최상위 카드 |

## 3. 현재 코드 반영 위치

```text
src/App.tsx
src/styles.css
```

### App.tsx

- `getGradeClass` 함수로 등급별 CSS 클래스를 결정한다.
- `CharacterCodexCard` 컴포넌트가 인물 도감 카드 UI를 담당한다.
- 주요 인물 목록에도 등급 클래스가 적용된다.

등급 클래스:

```text
grade-normal
grade-advanced
grade-rare
grade-legendary
```

### styles.css

- `.codex-card`가 카드 기본 프레임을 담당한다.
- `.card-aura`가 카드 주변 아우라를 담당한다.
- `.portrait-slot`은 임시 초상화 영역이다.
- 등급별 색상은 CSS 변수와 등급 클래스로 제어한다.

## 4. 현재 도감 카드 구성

인물 카드에는 아래 정보가 표시된다.

- 등급
- 캐릭터 ID
- 임시 초상화 슬롯
- 이름
- 역할
- 충성 수치
- 위험 수치
- 스킬 설명

## 5. 다음 확장 방향

1. 카드 이미지 생성 후 `portrait-slot`을 실제 이미지로 교체
2. 카드 클릭 시 상세 모달 표시
3. 도감 필터 추가
   - 전체
   - 일반
   - 고급
   - 희귀
   - 전설
4. 인물 외에도 이벤트/시설/아이템 카드에 같은 규칙 적용
5. 1000개 카드 생성 시 이 규칙을 템플릿으로 사용

## 6. 주의사항

- 카드 디자인 규칙은 게임의 시각적 정체성이므로 임의로 바꾸지 않는다.
- 등급 색상은 이후 밸런스/세계관 확장에 따라 추가될 수 있다.
- 실제 아트워크가 들어오기 전까지는 임시 초상화 슬롯을 유지한다.
