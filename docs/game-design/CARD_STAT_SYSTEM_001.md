# 카드 유동 스탯 시스템 001

작성일: 2026-06-16
상위 문서: `CARD_GAME_DESIGN_001.md`
데이터: `card-stat-schema.json`
연관: `character-growth-schema.json`(workTraits/skillTypes), `fusion-recipe-tree.json`(행운)
문서 성격: 고정 총량 재분배형(MMO식) 스탯 시스템 정형화

> 핵심: 카드의 1차 스탯 **총합은 등급·레벨로 고정**되며, **무슨 활동(작업)을 하느냐에 따라 총량 안에서 재분배**된다. **합성은 등급을 올리고 레벨을 초기화**한다. **스탯과 스킬은 별개**다.

---

## 1. 두 가지 성장 축 (서로 다른 개념)

```text
[합성 Fusion]  → 등급(rarity)을 올린다. 레벨을 1로 초기화한다. 스탯 총량(budget)이 점프한다.
[레벨업 Level] → 경험치로 오른다. budget을 조금씩 키운다. 증가분은 '활동'에 따라 분배된다.
```

- 합성은 **천장을 올리는 큰 도약**(대신 레벨 리셋), 레벨업은 **그 천장을 채우는 누적**이다.
- 즉 "합성으로 그릇을 키우고(레벨 리셋), 레벨업으로 다시 채운다."

---

## 2. 고정 총량(Budget) 모델

```text
budget(grade, level) = gradeBase[grade] + (level - 1) × levelStep[grade]
주어진 (등급, 레벨)에서  Σ(1차 스탯) = budget  으로 항상 일정.
```

| 등급 | gradeBase | levelStep | 최대레벨 | 최대 budget |
|---|---:|---:|---:|---:|
| N | 60 | 2 | 100 | 258 |
| R | 100 | 3 | 100 | 397 |
| SR | 150 | 4 | 100 | 546 |
| EP | 220 | 5 | 100 | 715 |
| L | 320 | 6 | 100 | 914 |

(v0 초안, 튜닝 대상. 등급 사다리는 N/R/SR/EP/L 기준.)

---

## 3. 1차 스탯 (유동 / 재분배 대상)

기본 3종은 사용자 지정(힘·민첩·지능), 나머지 5종은 제안.

| 스탯 | 코드 | 구분 | 의미 |
|---|---|---|---|
| 힘 | STR | 기본 | 물리 공격·제압·보호, 노동(채광) |
| 민첩 | DEX | 기본 | 속도·회피·행동 순서·수송/잠입 |
| 지능 | INT | 기본 | 기술 위력·은폐/해독/전략·스킬 효율 |
| 체력 | VIT | 제안 | HP·생존·피로 저항·탱킹 |
| 정확도 | ACC | 제안 | 명중·행동 성공 판정·치명 일부 |
| 친화 | CHA | 제안 | 협상/진정/서비스·동료화/회유 |
| 의지 | WIL | 제안 | 정신계/상태이상 저항·패닉 저항 |
| 행운 | LUK | 제안 | 합성 성공 보정·치명·희귀 드랍 |

> 제안 근거: `combat-actions.json`의 행동 계열(방어/비살상전투/비전투해결/은폐/조사/회복/시설제어/수송/심리/전략)을 전부 커버하려면 전투(STR/DEX/INT)뿐 아니라 생존(VIT)·판정(ACC)·사회(CHA)·정신(WIL)·운(LUK) 축이 필요하다. LUK은 이미 합성 가챠의 행운 보정과 연결된다.

### 파생(전투) 스탯 — 1차 스탯에서 계산, 직접 분배 불가
HP, 물리공격, 기술공격, 방어, 속도, 명중, 회피, 치명, 설득력, 저항. (공식은 `card-stat-schema.json` 참조)

## 4. 활동(작업) 기반 재분배 — 제로섬

```text
활동 1회 → statProfile 스탯이 +Δ
         → 동일한 총 Δ가 '잠기지 않은 비프로필 스탯'에서 비례 차감
         → 총량 budget은 불변 (제로섬)
```

예시(사용자 시나리오): **곡괭이질(채광)** = 힘을 쓰고 정확도가 필요한 노동 → `STR↑, ACC↑`, 대신 안 쓰는 INT·CHA 등이 조금씩 내려간다.

| 활동 | 오르는 스탯 |
|---|---|
| 채광(곡괭이질) | STR, ACC |
| 채혈 | ACC, DEX |
| 보관 | INT, STR |
| 수송 | DEX, VIT |
| 문서 | INT, ACC |
| 보안 | STR, VIT |
| 언론 대응 | CHA, WIL |
| 접수 | CHA, ACC |
| 던전 탐색 | DEX, INT |
| 전투 지휘 | STR, INT |

- 활동 목록은 `character-growth-schema.json`의 `workTraits`를 재활용 + 생활 노동(채광) 추가.
- 하한선(perStatFloor) 아래로는 내려가지 않는다.
- **레벨업 분배도 활동을 따른다**: 레벨업 증가분은 최근 활동 비중대로 분배되어, "무슨 일을 했느냐에 따라 레벨업 이득이 달라진다."

---

## 5. 잠금(Lock) 시스템

```text
스탯을 잠그면 활동 재분배 시 감소하지 않는다(차감 풀에서 제외 = 보호).
잠긴 동안에는 증가도 하지 않는다(고정).
잠금 슬롯 수는 등급에 비례: N1 · R1 · SR2 · EP2 · L3.
```

→ 원하는 스탯(예: 탱커는 VIT)을 잠가두고 다른 활동을 시켜도 핵심 스탯이 무너지지 않게 한다.

---

## 6. 합성 시 처리

```text
성공: 등급 +1, 레벨 → 1 초기화, budget = 새 등급 gradeBase로 점프.
분배: 이전 분배 '비율' 유지 + 새 budget으로 재정규화 (레벨만 리셋). ※ '완전 초기화'로 변경 가능
잠금: 초기화(해제)
스킬: 유지 (스탯과 별개, 합성으로 초기화 안 됨) — 확정 필요
```

---

## 7. 스탯 ≠ 스킬

| 구분 | 스탯 | 스킬 |
|---|---|---|
| 형태 | 연속 수치 | 이산적 능력 |
| 변동 | 활동으로 재분배(유동) | 별도 습득·레벨(고정 진행) |
| 역할 | 파생 전투 수치의 기반 | active/passive/ultimate/support/field/guild |
| 관계 | — | 스탯에 스케일하지만 재분배되지 않음 |

(skillTypes는 `character-growth-schema.json` 재활용)
