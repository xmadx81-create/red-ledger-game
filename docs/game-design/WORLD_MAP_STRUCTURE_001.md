# 월드맵 구조 001 (맵 · 대륙 · 섬 개념)

작성일: 2026-06-16
상위 문서: `CARD_GAME_DESIGN_001.md`, `CORE_LOOP_REDESIGN_001.md`
연관 데이터: `world-map.json`(스켈레톤), `stages.json`, `placement-locations.json`, `dungeon-types.json`
문서 성격: 지리 계층 **개념 정리만** (실제 대륙/섬 콘텐츠는 추후)

> 목적: 맵·대륙·섬·거점의 계층과 의미만 고정한다. 기존 스테이지(region)·배치 장소·던전을 이 계층 아래에 끼워 넣는다.

---

## 1. 4단계 지리 계층

```text
월드맵 (World)
 └─ 대륙 (Continent)        ← 큰 진행/테마/난이도 밴드 단위 (시즌·챕터 무대)
     └─ 섬 (Island)         ← 대륙 안 하위 권역. 해금·탐험의 기본 덩어리
         └─ 거점 (Site)     ← 실제 노드: 스테이지/던전/배치 장소/허브/상점
```

| 계층 | 의미 | 기존 연결 |
|---|---|---|
| 월드맵 | 헌혈센터 네트워크가 퍼진 세계 전체 | 허브(hub) 진입점 |
| 대륙 | 큰 권역. 난이도 밴드(Normal→Abyss)·세력/종족 테마·시즌 | `difficulty`, 명성 |
| 섬 | 대륙 내 존. 거점 묶음을 단위로 개방 | 스테이지/던전 그룹 |
| 거점 | 한 장소의 콘텐츠 노드 | `stages.json`(region), `placement-locations`(LOC-*), `dungeon-types`(DUN-*) |

---

## 2. 각 계층 정의

### 2.1 월드맵
- 게임의 최상위 화면. 어떤 대륙들이 열려 있는지 보여준다.
- 라이브 서비스: 시즌마다 새 대륙/섬을 추가해 무한 확장.

### 2.2 대륙 (Continent)
- **큰 단위의 진행·테마**. 기본은 난이도 밴드와 묶는다.
  - 예: 시작 대륙(Normal~Hard) → 다음 대륙(Elite) → … → Abyss 대륙.
- 종족/세력 권역으로도 활용 가능(인간권·이종족권·고대권 등).
- 스토리 챕터·시즌 목표의 무대.

### 2.3 섬 (Island)
- **대륙 안의 하위 권역(존)**. 스테이지·던전·배치 장소가 묶인 **해금 단위**.
- 한 섬을 클리어/명성 충족하면 다음 섬이 열리는 방식 → 점진적 개방.
- "한 입 크기" 콘텐츠 덩어리라 라이브 서비스에 적합.

### 2.4 거점 (Site / Node)
- 섬 안의 구체적 장소. 이미 있는 것들을 그대로 노드로 본다.
  - **허브/센터 본부**(운영·상점·도감)
  - **전투 스테이지**(`stages.json` STG-*)
  - **던전 게이트**(`dungeon-types.json` DUN-*)
  - **배치 육성 장소**(`placement-locations.json` LOC-*)

---

## 3. 개방(진행) 규칙

```text
거점 클리어 + 명성 충족 → 섬 개방
섬 클리어(주요 거점) + 난이도 도달 → 다음 대륙 개방
난이도 밴드: Normal → Hard → Elite → Nightmare → Abyss (대륙 단위)
```

명성(센터/인간 사회/암시장 등)과 스테이지 클리어가 개방 조건이 된다(`STAGE_REPUTATION_LEVEL_SYSTEM_001.md` 연계).

---

## 4. 식별 규칙(안)

```text
대륙:  CONT-001, CONT-002 ...
섬:    ISL-{대륙번호}-{섬번호}   예) ISL-001-02
거점:  기존 ID 재사용 — STG-*, DUN-*, LOC-*, HUB
```

이면: 특정 섬/야간/고난도 대륙에서만 은근한 긴장(VISUAL_TONE_GUIDE_001).
