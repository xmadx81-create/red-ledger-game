# 전투 규칙 방향성 001 (제안)

작성일: 2026-06-16
상위 문서: `CARD_GAME_DESIGN_001.md`
연관 데이터: `card-stat-schema.json`, `job-stat-profiles.json`, `combat-actions.json`, `combat-result-rules.json`, `mbti-synergy-rules.json`, `resources.json`(혈액)
문서 성격: 전투 시스템 **방향 제안**(확정 전). 세부 수치/규칙은 후속.

> 목적: 지금까지 만든 시스템(유동 스탯 → 파생 전투 스탯 → 직업 → 행동 카드 → 혈액 코스트 → 시너지)이 **실제 전투에서 어떻게 맞물리는지**의 방향을 잡는다.

---

## 1. 전제 (이미 정해진 것)

```text
- 파티: 영웅 카드 3~5장
- 자원: 혈액(매 턴 회복되는 카드 코스트)
- 행동 카드: combat-actions.json 9종(보호/제압/협상/은폐/증거회수/응급처치/봉쇄/회피/진정/후퇴)
- 스탯: 1차 8종 → 파생 전투 스탯(HP/물리·기술공격/방어/속도/명중/회피/치명/설득력/저항)
- 직업: 스탯으로 결정, 직업이 사용 가능한 스킬/행동 카드 풀을 좌우
- 멀티 장르: 같은 엔진, 승리 조건만 다름(전투/협상/잠입/수송/방어/보스)
```

---

## 2. 전투 진행 방향 (턴제 카드 배틀)

```text
1. 전투 시작: 파티 카드의 파생 스탯 산출, 속도(SPD) 순으로 행동 순서 결정
2. 매 턴: 혈액 일정량 회복 → 행동 카드 손패 드로우
3. 행동: 혈액을 소모해 행동 카드 사용 (어떤 카드를 쓸 수 있는지는 파티 구성/직업이 결정)
4. 판정: 명중(HIT) vs 회피(EVA), 치명(CRIT), 저항(RESIST) 굴림
5. 효과: combat-actions.json의 effects 수치를 스탯/직업/시너지로 보정해 적용
6. 시너지: 종족/성향/MBTI(mbti-synergy-rules) 조합 보너스 상시 적용
7. 승리 조건 충족 또는 전멸까지 반복
8. 결과: 클리어 랭크 판정(combat-result-rules) → 보상(경험치/재료/카드 드랍)
```

핵심: **행동 카드는 "무엇을 할지", 스탯/직업은 "얼마나 잘하는지"** 를 담당. 같은 `제압` 카드라도 STR/직업/시너지에 따라 위력이 다르다.

---

## 3. 직업 ↔ 전투 역할 매핑

| 직업 | 주력 파생 스탯 | 전투에서 |
|---|---|---|
| 마법사 | 기술공격 | 광역/기술 딜 |
| 전사 | 물리공격 | 단일 근접 딜 |
| 수호자 | HP/방어 | 탱킹·보호(ACT-PROTECT 강화) |
| 궁수 | 명중/치명 | 정확한 원거리 딜 |
| 도적 | 속도/회피/치명 | 선제·잠입(ACT-CONCEAL/EVADE) |
| 힐러 | 설득력/회복 | 응급처치(ACT-FIRST-AID) 강화 |
| 협상가 | 설득력 | 협상/진정(비전투 승리 조건) |
| 정신술사 | 저항/기술 | 상태이상·제어 |
| 술사 | 치명/행운 | 변수·크리·드랍 보정 |

→ 멀티 장르 스테이지에서 **승리 조건에 맞는 직업 조합**이 전략이 된다(협상전엔 협상가/힐러, 잠입전엔 도적/기록관).

---

## 4. 컨트롤 방식 — 핵심 결정 포인트

라이브 서비스 + 배치 육성과 어울리는 방식을 골라야 한다.

| 옵션 | 설명 | 장점 | 단점 |
|---|---|---|---|
| A. 수동 덱빌딩 | 매 턴 직접 카드 플레이 | 전략 깊이 | 반복 피로, 방치형과 충돌 |
| B. 오토 배틀 | 사전 전술 세팅 후 자동 진행 | 방치/대량 소화, 배치 철학과 일치 | 조작감 약함 |
| **C. 하이브리드(권장)** | 평소 오토, 고난도/보스는 수동 개입(스킬 타이밍) | 일상 반복은 편하게, 핵심 전투는 깊게 | 구현 복잡도 |

> **권장: C.** 일상 사냥(N 파밍·재료 수급)은 오토로 빠르게, 고난도·보스·PvP는 수동 개입. "합성에 미치게" 하려면 사냥 반복이 쾌적해야 하므로 오토 기본이 합리적.

> **구현 상태**: `battle-prototype`에 **수동 + 오토 토글(하이브리드 C)**이 구현됨. 오토는 검증된 휴리스틱(위급 회복 → 마법사 기술일격 → 전사 제압)으로 자동 진행. node 시뮬 + DOM 스텁 하니스로 50회 무작위 전투 무오류 완주 확인.

## 5. 판정/난수 방향

```text
명중률 = f(HIT 공격, EVA 방어)         → 빗나감 존재
치명 = f(CRIT) 확률, 피해 배수          → 한 방의 쾌감
저항 = f(RESIST) 상태이상/정신계 저항
행운(LUK) = 치명·드랍·합성에 광범위 보정 (전투-합성 경제 연결)
```

난수는 있되, 천장/보정으로 "운빨에만 좌우되지 않게"(윤리 가드레일).

---

## 6. 보상 → 경제 루프 연결

```text
전투 클리어 → N 카드/재료/경험치 드랍(card-acquisition.json) + 극히 드물게 잭팟 상위 카드
   → 합성 연료 축적 → 합성 도전(대부분 실패, 천장으로 진전)
      → 상위 카드/직업 강화 → 더 높은 난이도 → 반복
```

---

## 7. 후속 정형화 필요

```text
1. 컨트롤 방식 확정(A/B/C)            → 프로토타입에 하이브리드 C 구현됨
2. 혈액 회복량·카드 코스트 수치         → 프로토타입 v0 적용(시작4/턴+3/최대12)
3. 파생 스탯 → 실제 피해 공식 확정      → 프로토타입에 v0 공식 구현
4. 직업별 스킬 세트 정의               → 프로토타입에 직업 고유 스킬 1종씩 구현(전사/마법사/힐러)
5. 멀티 장르 승리 조건 수치화          → 설득전(협상 모드) 프로토타입 구현(공감/패닉/턴제한)
6. 클리어 랭크식(combat-result-rules)  → 미착수
7. 시너지 보정 수치(mbti/종족/성향)     → 프로토타입에 역할균형/인간연대 2종 v0 구현
```

> 프로토타입 검증(node 스텁, 오토 1000회): 직업 스킬·시너지 추가 후에도 승률 67.6%로 건강한 긴장 유지(스킬이 혈액·쿨다운을 소모해 자연 균형).

> 멀티 장르 검증(설득전): 협상형 파티(고 설득력)는 100% 클리어, 전사 파티(저 설득력)는 0% — 스테이지 유형에 맞는 직업 구성이 승패를 가르는 멀티 장르 설계가 실제로 작동. 섬멸전 ~70%와 함께 모드별 무오류 완주.

> 본 문서는 방향 제안이다. 컨트롤 방식이 정해지면 전투 상세 규칙 문서로 확정한다.
