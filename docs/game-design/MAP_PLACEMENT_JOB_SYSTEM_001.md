# 맵 배치 → 직업 육성 시스템 001

작성일: 2026-06-16
상위 문서: `CARD_GAME_DESIGN_001.md`, `CARD_STAT_SYSTEM_001.md`
데이터: `placement-locations.json`, `job-stat-profiles.json`, `card-stat-schema.json`, `jobs.json`
문서 성격: 배치 기반 육성 + 스탯 기반 자유 직업 체계 정형화

> 핵심 루프: **맵의 장소에 카드 배치 → 그 장소의 작업 수행 → 스탯 재분배 → 스탯 분포가 직업 후보를 자동 산출 → 유저가 후보 중 선택.** 종족과 무관하게, 배치 육성만으로 어떤 카드든 원하는 직업이 된다.

---

## 1. 왜 이렇게 가는가

카드 게임이라 MMO처럼 캐릭터 하나하나를 직접 조종하며 퀘스트를 따라갈 수 없다. 대신 **맵에 배치**하는 행위로 육성 방향을 정한다.

```text
MMO: 캐릭터를 데리고 다니며 직접 행동 → 스탯 성장
본작: 카드를 장소에 배치 → 장소의 작업이 대신 수행 → 스탯 성장
```

이 방식은 "항상 할 일이 있는" 라이브 서비스(상시 배치·누적)와도 맞물린다.

---

## 2. 핵심 루프

```text
1. 맵에서 장소 선택 (또는 던전 게이트)
2. 카드를 배치 (장소별 수용 인원 capacity)
3. 배치 시간이 쌓이며 장소의 작업 틱 발생
4. 작업 프로필에 따라 스탯 재분배 (card-stat-schema 제로섬 규칙)
5. 스탯 분포가 직업 후보 조건을 충족하면 후보 등록
6. 유저가 직업 후보 중 선택 → 직업 확정(스킬/역할/승급로 부여)
7. 다른 방향으로 키우고 싶으면 재배치 → 스탯 이동 → 전직 가능
```

---

## 3. 맵 / 장소 (placement-locations.json)

각 장소는 특정 작업을 수행하며 정해진 스탯을 올린다(나머지는 제로섬 차감).

| 장소 | 작업 | 올리는 스탯 | 가는 직업 |
|---|---|---|---|
| 마법 연구실 | 룬 해독 | INT, WIL | 마법사 |
| 훈련장 | 전투 훈련 | STR, VIT | 전사 / 수호자 |
| 사격장 | 정밀 사격 | DEX, ACC | 궁수 |
| 잠입 통로 | 은밀 기동 | DEX, INT | 도적 |
| 채혈/의무실 | 채혈·치료 | INT, CHA | 힐러 / 의무관 |
| 보안실 | 경비 | STR, VIT | 수호자 / 보안요원 |
| 상담 로비 | 접수·상담 | CHA, ACC | 협상가 |
| 기록실 | 문서·해독 | INT, ACC | 기록관 |
| 수송 차고 | 수송 | DEX, VIT | 수송원 |
| 광산 | 채광(곡괭이질) | STR, ACC | 전사/궁수(+자원) |
| 행운의 방 | 의식·점복 | LUK, WIL | 술사 |
| 던전 게이트 | 던전 파견 | 가변 + 경험치/재료 | 전투 전반 |

- 한 카드를 여러 장소에 **순차 배치**해 복합 스탯(예: INT+CHA = 힐러)을 만든다.
- 던전 파견도 배치의 한 형태(경험치·재료·드랍 + 가변 스탯 훈련). `dungeon-types.json` 연계.

## 4. 스탯 → 직업 후보 산출 (job-stat-profiles.json)

```text
판정: 각 직업의 dominantStats 값이 budget 대비 minShare 이상이면 조건 충족.
      직업의 모든 dominantStats 조건을 만족하면 '후보'로 등록.
선택: 후보가 여럿일 수 있고, 유저가 하나를 고른다.
전직: 스탯이 유동적이라 재배치로 스탯을 바꾸면 다른 직업 후보로 전환 가능.
```

직업 아키타입(JOBX-*, 종족 무관):

| 직업 | 주 스탯 | minShare | 역할 |
|---|---|---:|---|
| 마법사 | INT | 30% | 기술공격 딜러 |
| 전사 | STR | 30% | 근접 딜러 |
| 수호자 | VIT, STR | 22% | 탱커/방어 |
| 궁수 | DEX, ACC | 22% | 원거리 딜러 |
| 도적 | DEX | 30% | 잠입/회피/치명 |
| 힐러 | INT, CHA | 20% | 회복/지원 |
| 술사 | LUK | 25% | 변수/치명/드랍 |
| 정신술사 | WIL, INT | 20% | 정신계 제어/디버프 |

기존 운영 직업(JOB-*, jobs.json)도 스탯 조건으로 연결: 의무관·보안요원·협상가·기록관·수송원.

> jobs.json의 `mainStats`는 옛 시뮬 자원 기준이므로, 카드 스탯 기반 후보 판정은 본 시스템의 `dominantStats`를 사용한다.

---

## 5. "종족 무관 자유 직업"의 의미

```text
직업 후보는 '스탯'으로만 판정한다. 종족 제한이 없다.
→ 어떤 종족의 어떤 캐릭터라도 마법 연구실에 배치해 INT를 키우면 '마법사' 후보가 뜬다.
→ 골렘 마법사, 호빗 전사, 뱀파이어 힐러 모두 가능.
→ 종족은 '시작 스탯 성향/종족 버프'로만 차별화(racePromotionBuffPolicy 참조), 직업을 막지 않는다.
```

이것이 "세상에 없던 개념일 수도 있는" 자유 직업 + 유동 스탯의 결합이다.

---

## 6. 기존 시스템과의 연결

```text
card-stat-schema.json : 스탯 재분배 규칙(제로섬)·budget·잠금 — 배치가 이 규칙을 구동
dungeon-types.json    : 던전 게이트 배치 대상 + 재료/경험치 공급
jobs.json             : 운영 직업 정의(승급로/유니크버프) — 스탯 후보와 연결
character-growth-schema: workTraits/skillTypes/승급 — 활동·스킬 출처
```

---

## 7. 결정/미해소

```text
1. 직업 확정의 가역성: 전직 자유 vs 비용·쿨다운 부과
2. 한 카드가 동시에 1직업 vs 서브직업 허용
3. 배치 누적 속도(틱)·capacity·오프라인 누적 한도 (라이브 서비스 밸런스)
4. 직업 스킬 세트 정의(직업별 active/passive/ultimate)
5. 종족 버프가 직업 후보 minShare에 보정을 줄지 (예: 위저드 종족은 마법사 문턱↓)
6. 등급 사다리 통일(기존 미해소 이슈)과 budget·후보 문턱 정렬
```

> 본 문서·데이터는 디자인(이미지) 이전의 데이터/밸런스 정형화 산출물이다.
