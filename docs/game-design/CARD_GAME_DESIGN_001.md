# 헌혈의 집 — 카드 게임 기획서 001

작성일: 2026-06-15
문서 성격: **마스터 기획서 (장르 전환 기준 문서)**

> 이 문서는 기존 "시뮬레이션 경영" 기획을 대체한다. 모티브와 컨셉(*헌혈의 집*, 밝은 표면·은근한 이면)은 유지하되, **장르를 수집형 카드 배틀 RPG로 전환**한다. 기존 운영 시뮬레이션 문서(`00_game-design-index.md`, `01_core-loop.md`, `MVP_7DAYS_PLAN.md`, 엔딩 관련)는 본 문서 기준으로 **폐기 또는 재해석** 대상이다.

## 0. 장르 전환 배경

처음에는 *헌혈의 집*을 모티브로 한 **운영 시뮬레이션**으로 시작했다. 그러나 개발을 진행하며 "수집할 캐릭터 → 전투 → 레벨링 → 등급/아우라"를 계속 추가하다 보니 게임의 무게중심이 자연스럽게 **카드 수집·전투형 RPG**로 이동했다.

따라서 기획을 정리한다.

```text
유지: 모티브(헌혈의 집), 톤(밝은 표면·은근한 이면), 세계관(공생/비밀), 카드 시스템
전환: 핵심 장르를 "운영 시뮬레이션" → "수집형 카드 배틀 RPG"로 변경
폐기: 7일 운영 루프, 엔딩/최종 보고서, 가문 중심 다크 누아르 경영 자원
```

## 1. 한 줄 컨셉

```text
밝고 명랑한 동네 「헌혈의 집」을 거점으로,
인간과 이종족이 공생하기 위한 혈맹 카드를 모으고,
카드를 레벨업·합성·강화해 끝없이 더 강한 카드를 만들어 가는,
엔딩 없는 실시간 온라인 성장형 카드 게임.
```

> **장르 정체성 핵심**: 이 게임은 "기간 한정 시나리오"가 아니라 **실시간 온라인 라이브 서비스**를 모티브로 한다. 종료(엔딩)가 없고, 유저는 **카드를 계속 더 좋게 만들기 위해 끊임없이 액션을 취한다**. 전투는 그 성장을 위한 수단이며, **카드 성장·합성·강화 루프 자체가 지속 동기**다. (§4-A 참조)

- **표면**: 깨끗하고 따뜻한 지역 헌혈센터. 봉사·회복·일상·커뮤니티.
- **이면**: 혈액은 이종족(뱀프 등)의 생존선. 센터는 인간–이종족 공생 협약의 조용한 거점이다. 암거래, 오해한 인간 세력, 적대 분파가 사건을 일으킨다.
- **톤 원칙**: `밝은 표면, 은근한 이면.` 다크 요소는 야간 이벤트/특정 스테이지/스토리에서만 제한적으로 드러난다. (`VISUAL_TONE_GUIDE_001.md` 준수)

## 2. 게임 정체성

| 항목 | 내용 |
|---|---|
| 제목 | 헌혈의 집 |
| 장르 | 수집·성장형 카드 게임 (덱/파티 빌딩 + 턴제 카드 전투 + 카드 합성/강화) |
| 서비스 형태 | **실시간 온라인 라이브 서비스 모티브**. 상시 접속·지속 운영·기간 한정 없음 |
| 진행 구조 | **엔딩 없는 영구 성장형.** 스테이지/난이도/시즌은 반복되는 성장 무대일 뿐 종료점이 아님 |
| 핵심 동기 | **카드를 레벨업·합성·강화해 더 좋은 카드를 만드는 무한 성장 루프** (전투는 이를 위한 수단) |
| 핵심 재미 | ① 종족 영웅 카드 수집 ② 카드 합성/강화로 상위 카드 제작 ③ 덱·파티 시너지 설계 ④ 멀티 장르 스테이지 공략 |
| 핵심 자원 | 혈액(카드 코스트), 명성, 성장·합성 재화 |

## 3. 카드 분류 (3계층)

이 게임의 모든 콘텐츠는 카드로 표현한다.

### 3.1 영웅 카드 (Hero Card) — 수집 대상
- 도감의 본체. 종족·직업·역할·성향·등급(N/R/SR/EP/L)·아우라를 가진다.
- **다양성 기반**: 기본 종족 14종 + **모든 종족 쌍의 범용 하이브리드(91종)**. 일반(N) 카드가 압도적 다수. 상세 → `RACE_HYBRID_FUSION_SYSTEM_001.md`
- 데이터 소스: `hero-roster-by-race.json`, `enemy-hero-roster.json`, `character-card-registry.json`
- 전투 스탯: **1차 스탯(힘/민첩/지능 + 체력/정확도/친화/의지/행운)에서 HP·공격·방어·속도 등이 파생**. 1차 스탯은 고정 총량 안에서 활동에 따라 재분배되는 유동형. 상세 → `CARD_STAT_SYSTEM_001.md`
- 성장: 레벨/경험치/스킬/장비/승급(전직). (`CORE_LOOP_REDESIGN_001.md`, `character-growth-schema.json`)
- 카드 식별: `cardUid = CARD-{CHARACTER_ID}-{RARITY}-{SERIAL}` (`CARD_PRODUCTION_PIPELINE_001.md`)

### 3.2 행동 카드 (Action Card) — 전투 플레이의 핵심
- 매 턴 손에 들고 혈액을 소모해 사용하는 카드. **덱빌딩의 중심.**
- 데이터 소스: `combat-actions.json` (이미 9종 정의됨)

| actionId | 이름 | 계열 | 카드 게임에서의 역할 |
|---|---|---|---|
| ACT-PROTECT | 보호 | 방어 | 아군/방문자 피해 차단 |
| ACT-SUBDUE | 제압 | 비살상 전투 | 위협 무력화(비살상) |
| ACT-NEGOTIATE | 협상 | 비전투 | 적대/패닉 감소, 신뢰 상승 |
| ACT-CONCEAL | 은폐 | 비밀 유지 | 비밀 노출 감소 |
| ACT-EVIDENCE | 증거 회수 | 조사 | 해금 조건·단서 확보 |
| ACT-FIRST-AID | 응급 처치 | 회복 | 생존/안전 회복 |
| ACT-LOCKDOWN | 보안 봉쇄 | 시설 제어 | 위협 확산 차단 |
| ACT-EVADE | 회피 기동 | 아케이드/수송 | 추격 회피 |
| ACT-CALM | 진정 안내 | 서비스/심리 | 패닉 확산 차단 |
| ACT-RETREAT | 후퇴 | 전략 | 피해 최소화(보상 감소) |

> 행동 카드는 "어떤 영웅이 쓸 수 있는가(`usableBy`)"가 정해져 있어, **파티 구성 = 사용 가능한 행동 카드 풀**이 된다. 이것이 덱빌딩과 직결된다.

### 3.3 자원: 혈액 (Blood) — 카드 코스트
- 매 턴 회복되는 코스트 자원(타 게임의 마나/에너지 역할).
- 기존 `resources.json`의 `혈액 재고/혈액 수요`를 **전투 코스트 경제**로 재해석.
- 혈액이 부족하면 강한 카드를 못 쓰고, 과소비하면 다음 턴이 약해지는 운영 긴장감.

## 4. 핵심 플레이 루프

```text
1. 「헌혈의 집」 본부에서 사건/스테이지 선택
2. 파티(영웅 카드) 편성 + 행동 카드 덱 점검
3. 스테이지 진입 (전투/협상/잠입/수송 등 모드별 승리 조건)
4. 턴제 카드 배틀: 혈액을 소모해 행동 카드 사용, 시너지 발동
5. 클리어 → 랭크 판정 → 보상(경험치/재화/승급 재료/신규 카드)
6. 영웅 카드 성장·승급, 행동 카드 덱 개선
7. 명성 상승 → 새 스테이지/난이도/종족 해금
8. 더 높은 난이도 재도전, 시즌 목표 추구 (반복)
```

엔딩은 없다. (`CORE_LOOP_REDESIGN_001.md` 준수) 표현은 `스테이지 결과 / 랭크 판정 / 다음 스테이지 개방 / 시즌 진행도`를 사용한다.

## 4-A. 지속 성장 엔진 (라이브 서비스 핵심)

> 이 게임의 **심장**. §4의 전투 루프는 이 엔진에 연료(재료·재화·카드)를 공급하기 위한 수단이다. 유저가 게임을 계속하는 이유는 "사건 해결"이 아니라 **"내 카드를 계속 더 좋게 만드는 것"** 이다.

### 핵심 명제
```text
유저는 항상 "다음에 더 좋은 카드를 만들기 위한 액션"을 가지고 있어야 한다.
할 일이 떨어지는 순간이 없어야 하고, 그 액션은 명확한 성장 체감으로 보상되어야 한다.
```

### 카드를 더 좋게 만드는 4가지 축

| 축 | 행위 | 결과 | 재료 공급원 |
|---|---|---|---|
| 레벨업 | 경험치 투입 | 스탯 상승, 스킬 강화 | `DUN-EXP`, 전투 보상 |
| 합성(Fusion) | 카드 + 카드/재료 합치기 | 상위 등급·신규 상위 카드 생성 | 중복 카드, 합성 재료 |
| 강화(Enhance) | 강화 재료 투입 | 같은 카드의 능력치 강화 | `DUN-ITEM`, 강화석 |
| 승급/전직(Promotion) | 승급 재료 투입 | 등급·역할 상위 전환 | `DUN-PROMOTION`, 종족 결정 |

> **합성이 이 게임의 "가챠"다.** 카드 뽑기(pull)는 없다. 합성은 **확률 + 행운**으로 판정되어 대부분 실패하고, 아주 가끔 상위 등급으로 도약한다. 약한 카드·중복 카드는 버려지지 않고 합성 재료가 된다. 단 실패만 반복시키지 않도록 **천장형 누적 보정**으로 꾸준한 진전을 보장한다(윤리 가드레일). 상위 카드가 나오면 **새 `cardUid`를 발급**한다. 일반(N) 카드는 출석/로그인·드랍으로 **무료 정기 공급**해 합성 재료가 끊기지 않게 한다. 상세 규칙·확률식·공급 모델 → `RACE_HYBRID_FUSION_SYSTEM_001.md`

### 무한 성장 루프
```text
전투/던전으로 재료·재화·중복 카드 확보
   → 카드 레벨업·강화·합성·승급
      → 더 강한 카드/덱 완성
         → 더 높은 난이도·신규 스테이지 도전 가능
            → 더 좋은 재료 확보
               → (반복, 종료 없음)
```

### 라이브 서비스 / 실시간 온라인 모티브
```text
상시 접속형: 세계와 콘텐츠가 계속 돌아간다(기간 한정 시나리오 아님).
지속 공급: 일일/주간/시즌 콘텐츠로 성장 재료가 끊기지 않게 한다.
항상 할 일: 접속하면 "지금 할 수 있는 카드 강화 액션"이 항상 존재한다.
```

> **윤리 가드레일 유지**(`CORE_LOOP_REDESIGN_001.md`): 지속 동기는 만들되, 과금 압박·강제 접속 피로도·확률 조작·노가다 강요는 금지. "계속 하고 싶게"는 OK, "안 하면 손해 보게 만드는" 설계는 지양.

## 5. 전투 시스템 (턴제 카드 배틀)

### 5.1 기본 규칙
- 플레이어: 영웅 카드 3~5장으로 구성된 **파티**.
- 매 턴: 행동 카드 손패를 뽑고(드로우), **혈액**을 소모해 카드를 낸다.
- 적: `enemy-hero-roster.json` 기반 적 영웅/보스가 자체 패턴으로 행동.
- 승패: 모드별 승리 조건 충족 또는 전멸.

### 5.2 스탯·효과
- 행동 카드 효과는 `combat-actions.json`의 `effects`를 그대로 전투 수치로 사용한다.
  - 예: `보호` → 안전도+, `협상` → 패닉−·신뢰+, `회피 기동` → 추격위험−.
- 즉, **전투/협상/잠입/수송이 같은 카드 프레임 위에서 "승리 조건만 다르게"** 돌아간다.

### 5.3 클리어 랭크
- 클리어 품질에 따라 랭크 판정(예: S/A/B/C). 후퇴(`ACT-RETREAT`)는 `clearRank-1`처럼 랭크를 깎는다.
- 랭크는 보상량·해금에 영향. (`combat-result-rules.json` 활용)

## 6. 덱 / 파티 시너지

수집형 카드 게임의 깊이는 시너지에서 나온다. 기존 데이터가 그대로 시너지 축이 된다.

| 시너지 축 | 데이터 소스 | 효과 예시 |
|---|---|---|
| 종족 시너지 | `hero-roster-by-race.json` | 동일/상성 종족 편성 시 보너스 |
| 성향 아우라 | `card-style-system.json` | GOOD/NEUTRAL/EVIL 조합 보너스·페널티 |
| 직업 조합 | `jobs.json` | 탱커+힐러+딜러 균형 보너스 |
| MBTI 시너지 | `mbti-synergy-rules.json` | 성격 조합에 따른 협동 보정 |
| 교대/근무 | `shift-rotation-rules.json` | 야간/주간 스테이지에서 근무 적합도 보너스 |
| 간호사 NPC | `nurse-npcs.json` | 센터 상주 NPC가 전투 외 보조 효과 |

> 핵심 설계: **"좋은 카드를 모으는 것"만으로 강해지지 않고, "어떻게 조합하느냐"가 강함을 결정한다.** 수집과 전략을 동시에 자극.

## 7. 멀티 장르 스테이지

모든 스테이지가 전투는 아니다. 같은 카드 프레임, 다른 승리 조건. (`CORE_LOOP_REDESIGN_001.md`, `dungeon-types.json`)

| 모드 | 승리 조건 | 핵심 행동 카드 | 던전 매핑 |
|---|---|---|---|
| 집단 전투 | 적 무력화/생존 | 제압·보호·응급처치 | DUN-EXP, DUN-GUILD-WEEKLY |
| 설득/협상 | 신뢰↑·패닉↓ 도달 | 협상·진정 안내 | (오해한 인간 세력) |
| 잠입/조사 | 단서/증거 확보 | 은폐·증거 회수 | DUN-PUZZLE |
| 야간 수송 아케이드 | 추격 회피·수송 성공 | 회피 기동 | DUN-ARCADE |
| 방어전 | 시설 안정·확산 차단 | 보안 봉쇄·보호 | (센터 방어전) |
| 보스전 | 보스 패턴 공략 | 복합 | DUN-FORBIDDEN |

성장/재화 던전: `DUN-EXP`(경험치), `DUN-GOLD`(재화), `DUN-ITEM`(장비), `DUN-PROMOTION`(승급 재료) 등은 카드 성장 공급원으로 유지.

## 8. 수집 · 성장 · 등급

### 8.1 등급 / 아우라 (`card-style-system.json` 그대로 유지)
| 등급 | 코드 | 테두리 | 역할 |
|---|---|---|---|
| 일반 | N | 무채색 | 기반층 |
| 고급 | R | 초록 | 역할 특화 |
| 희귀 | SR | 파랑 | 전투/운영 핵심 |
| 영웅 | EP | 보라 | 메인 스토리 |
| 전설 | L | 황금 | 시즌 핵심 |

성향 아우라: 선(푸름)/악(검붉음)/중립(보라)/전설(황금).

> **세분화**: 5개 메이저 등급은 각각 내부 ★단계(N ★1~5 … L ★1~10)를 가진다. 등급 내 ★ 상승은 강화로, **다음 메이저 등급 도약은 합성(저확률)으로만** 일어난다. 따라서 **전설 도달은 의도적으로 매우 오래** 걸린다. 상세 → `RACE_HYBRID_FUSION_SYSTEM_001.md` §4.

### 8.2 성장 — 4축 (상세는 §4-A 지속 성장 엔진)
- **레벨업**: 경험치로 budget(스탯 총량)을 키움. **합성과 별개 개념**이며, 증가분은 활동에 따라 분배. 합성 시 레벨은 초기화된다. 상세 → `CARD_STAT_SYSTEM_001.md`
- **강화(Enhance)**: 강화 재료로 같은 카드의 능력치를 끌어올림.
- **합성(Fusion)**: 중복 카드·하위 카드·합성 재료를 합쳐 **상위 등급/신규 상위 카드를 생성**. (진행의 정점)
- **승급/전직(Promotion)**: `DUN-PROMOTION` 재료로 등급/역할을 상위 전환.
- 합성·승급으로 상위 카드가 생기면 동일 캐릭터라도 **새 cardUid**를 발급한다(`CARD_PRODUCTION_PIPELINE_001.md` 규칙 유지).

### 8.3 수집/획득 윤리
- 과금 압박·확률 조작·강제 접속형 설계는 금지(`CORE_LOOP_REDESIGN_001.md`). 명확한 보상·수집 욕구·전략 실험 중심.

## 12. MVP 범위 (카드 게임 1차)

기존 `src/web-mvp`를 확장한다.

```text
포함:
- 카드 스튜디오(있음): 영웅 카드 렌더 + cardUid 검증
- 도감 화면(있음): 수집 목록
- 전투 프로토타입(신규): 파티 3장 + 행동 카드 5종 + 혈액 코스트 + 1개 모드(집단 전투)로 1전투 검증
- 스테이지 1개 + 클리어 랭크 + 보상 1회
- 혈액 코스트 경제 최소 구현

제외(후속):
- 승급/전직 풀 구현
- 멀티 모드 전부
- 시즌/명성 메타 풀 구현
- 가챠/상점
```

## 13. 결정 사항

### 13.1 확정된 방향
```text
- 서비스: 실시간 온라인 라이브 서비스 모티브, 상시 접속형 (기간 한정 아님)
- 진행: 엔딩 없는 영구 성장형 (런 단위 리셋/로그라이크 아님)
- 핵심 동기: 카드 레벨업·합성·강화·승급으로 더 좋은 카드를 만드는 무한 루프
- 다양성: 일반(N) 카드가 압도적 다수. 기본 종족 14종 + 모든 종족 쌍 하이브리드(91종)
- 가챠: 카드 뽑기(pull) 없음. "합성 가챠"만 존재 (합성 시 확률+행운 판정)
- 합성: 대부분 실패, 드물게 상위 등급 도약. 단 천장형 누적 보정으로 진전 보장
- 등급: N~L 메이저 5단계 + ★ 내부 세분화. 전설 도달은 의도적 장기화
- 공급: 일반 카드 무료 정기 공급(출석/로그인/드랍)
- 윤리: 과금 압박·강제 접속 피로도·확률 조작·노가다 강요 금지
- 상세: RACE_HYBRID_FUSION_SYSTEM_001.md
```

### 13.2 아직 확정 필요한 노브
```text
1. 전투 형태: (A) 1:N 파티 vs 적 / (B) 1캐릭터 덱 빌딩 → 기본안 (A) 파티형
2. 손패/덱: 행동 카드 "공용 풀" vs "영웅별 고정 스킬" → 기본안 영웅이 풀을 결정하는 혼합
3. 혈액 코스트: 매 턴 고정 회복 vs 누적형 → 기본안 고정 회복
4. 수집 방식: 가챠 vs 스테이지 드랍 vs 제작/합성 → 비과금 압박 원칙상 드랍/합성 우선
5. 온라인 범위: (a) 싱글 + 라이브 서비스(랭킹/시즌/길드) vs (b) 실시간 PvP 포함
   → 기본안 (a). 실시간 PvP는 MVP 이후 검토 (실시간 동기화 비용 큼)
6. 멀티플레이: 길드 협동 던전(`DUN-GUILD-WEEKLY`)을 비동기로 둘지 실시간으로 둘지
```
