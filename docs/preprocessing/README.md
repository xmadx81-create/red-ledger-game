# 전처리 단계 기록소

이 문서는 게임 프로젝트 `red-ledger-game`의 개발 준비 단계, 즉 **전처리 단계**를 기록하기 위한 기준 문서입니다.

## 목적

전처리 단계의 목적은 본격적인 게임 제작에 들어가기 전에 협업 도구, 기록 체계, 저장소 구조, 작업 로그 방식을 먼저 고정하는 것입니다.

이 단계에서 우선 준비할 연동 도구는 다음 순서로 진행합니다.

1. GitHub
2. Google Drive
3. Google Sheets
4. Figma
5. Google Calendar
6. Gmail

## 현재 기준 저장소

- GitHub 계정: `xmadx81-create`
- 저장소: `xmadx81-create/red-ledger-game`
- 저장소 공개 범위: Private
- 기본 브랜치: `main`
- 전처리 기록 경로: `docs/preprocessing/`

## 문서 구성

| 파일 | 역할 |
|---|---|
| `README.md` | 전처리 단계의 목적과 기준 설명 |
| `CHECKLIST.md` | 도구별 준비 상태 체크리스트 |
| `WORK_LOG.md` | 실제 진행 이력 및 결정사항 기록 |

## 운영 원칙

- 중요한 결정사항은 반드시 `WORK_LOG.md`에 남깁니다.
- 준비해야 할 항목은 `CHECKLIST.md`에 먼저 추가한 뒤 처리합니다.
- 도구 연동, 권한 설정, 폴더 구조 변경, 문서 생성은 모두 작업 로그에 기록합니다.
- 본격 개발 단계로 넘어가기 전, 전처리 단계 문서를 한 번 정리하고 확정합니다.
