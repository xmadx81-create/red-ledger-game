# 1차 MVP 화면 목록

이 문서는 `red-ledger-game` 1차 MVP에서 우선 설계할 화면 목록을 정의합니다.

|---|---|---|---|---|---|

## 핵심 화면 우선순위

가장 먼저 설계해야 할 화면은 다음 5개입니다.

1. `SCR-003` 메인 운영 화면
2. `SCR-005` 이벤트 발생 화면
3. `SCR-006` 선택 결과 화면
4. `SCR-002` 일차 시작 보고 화면
5. `SCR-008` 일차 종료 보고 화면

## 화면 흐름 초안

```text
타이틀 화면
→ Day 시작 보고
→ 메인 운영 화면
→ 이벤트 발생
→ 선택지 선택
→ 선택 결과
→ 메인 운영 화면 복귀
→ 일차 종료 보고
→ 다음 Day
→ 7일 최종 평가
```

## 화면 설계 시 확인할 것

- 현재 Day가 명확히 보이는가?
- 혈액 재고, 가문 자금, 가문 위신, 인간 신뢰, 언론 노출, 조직 불안, 보안 등급, 가문 만족도가 파악되는가?
- 선택지의 결과가 예측 가능하되 완전히 노출되지는 않는가?
- 위험 자원이 상승할수록 긴장감이 생기는가?
- 텍스트가 너무 길어 화면을 압도하지 않는가?
- 모바일 세로 화면에서 조작 가능한가?

## Figma 연결 기준

각 화면은 Figma에서 같은 `화면ID`를 프레임명 앞에 붙입니다.

예시:

```text
SCR-003_Main_Operation_Mobile
SCR-005_Event_Mobile
SCR-006_Result_Mobile
```

GitHub 문서, Google Sheets의 `08_UI화면목록`, Figma 프레임명이 같은 화면ID를 공유해야 합니다.
