# 본격 기획 단계 작업 로그 부록 - 첫 화면 흐름

## 작업 주제

첫 화면 흐름 설계

## 진행 내용

1. `docs/ui-ux/FIRST_SCREEN_FLOW.md`를 생성했다.
2. 타이틀 화면부터 Day 1 메인 운영 화면까지의 흐름을 정의했다.
3. 프롤로그 문서, 부임 통지, 봉인 보고서, Day 1 시작 보고, 핵심 자원 튜토리얼, 첫 선택지, 선택 결과, 메인 운영 화면 진입 구조를 정리했다.
4. 첫 화면에서 모든 세계관을 설명하지 않고, 표면 업무와 이면 보고의 불일치를 통해 게임의 정체성을 전달하도록 설계했다.
5. Figma 프레임명 기준을 정리했다.

## 결정 사항

- 첫 인상은 과장된 공포보다 차가운 행정적 위화감으로 잡는다.
- 초반에는 핵심 자원을 일부만 노출하고 나머지는 Day 진행에 따라 해금한다.
- 첫 선택지는 정석, 가문 우선, 은폐/중용의 3방향을 제공한다.
- 첫 화면 흐름은 Day 1 메인 운영 화면으로 자연스럽게 이어져야 한다.
