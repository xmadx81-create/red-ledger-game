# Figma 화면 설계 기준

이 문서는 `red-ledger-game` 프로젝트의 Figma 화면 설계 기준을 정의합니다.

## 목적

Figma는 게임의 화면 구조, UI 흐름, 와이어프레임, 시각 톤을 설계하는 도구입니다.

GitHub 문서가 기준과 기록을 담당한다면, Figma는 사용자가 실제로 보게 될 화면과 조작 흐름을 시각화하는 역할을 맡습니다.

## 권장 Figma 파일명

```text
red-ledger-game_적혈의장부_UIUX
```

## Figma 페이지 구조

```text
00_Cover
01_Project_Brief
02_User_Flow
03_Wireframe_Mobile
04_Wireframe_PC
05_Game_Screens
06_Components
07_Visual_References
99_Archive
```

## 페이지별 역할

| 페이지 | 역할 |
|---|---|
| `00_Cover` | 프로젝트명, 버전, 상태, 최종 수정일 표시 |
| `01_Project_Brief` | 게임 콘셉트, 장르, 1차 MVP 기준 요약 |
| `02_User_Flow` | 시작 → 운영 → 이벤트 → 결과 흐름 정리 |
| `03_Wireframe_Mobile` | 모바일 기준 와이어프레임 |
| `04_Wireframe_PC` | PC/태블릿 기준 와이어프레임 |
| `05_Game_Screens` | 실제 게임 화면별 상세 설계 |
| `06_Components` | 버튼, 카드, 게이지, 팝업, 선택지 컴포넌트 |
| `07_Visual_References` | 색감, 분위기, 레이아웃 참고 자료 |
| `99_Archive` | 폐기/보류된 화면안 보관 |

## 기본 화면 목록

1차 MVP 기준으로 우선 설계할 화면은 다음과 같습니다.

| 화면ID | 화면명 | 목적 | 우선순위 |
|---|---|---|---|
| `SCR-001` | 타이틀 화면 | 게임 시작, 불러오기, 설정 진입 | 높음 |
| `SCR-002` | 일차 시작 보고 화면 | Day별 상황 요약 | 높음 |
| `SCR-003` | 메인 운영 화면 | 핵심 자원과 오늘의 업무 확인 | 높음 |
| `SCR-004` | 자원 상태 상세 화면 | 혈액 재고, 자금, 위신 등 상세 확인 | 높음 |
| `SCR-005` | 이벤트 발생 화면 | 사건 설명과 선택지 제시 | 높음 |
| `SCR-006` | 선택 결과 화면 | 선택에 따른 자원 변화 확인 | 높음 |
| `SCR-007` | 인물/조직 관리 화면 | 주요 인물 상태와 조직 불안 확인 | 중간 |
| `SCR-008` | 일차 종료 보고 화면 | 하루 결과 정리 | 높음 |
| `SCR-009` | 7일 최종 평가 화면 | 엔딩 분기와 최종 결과 | 높음 |
| `SCR-010` | 설정 화면 | 사운드, 텍스트 속도, 저장 관련 설정 | 낮음 |

## 모바일 기준

초기 기준은 모바일 세로 화면입니다.

권장 기준:

```text
390 x 844
```

또는 Figma 기본 프리셋 기준:

```text
iPhone 14 / 15 계열 세로 프레임
Android Large 세로 프레임
```

## PC/태블릿 기준

PC 또는 태블릿 대응은 2차 기준으로 둡니다.

권장 기준:

```text
1280 x 720
1440 x 900
```

## 화면 설계 원칙

- 플레이어가 현재 Day, 핵심 자원, 위험도를 즉시 파악할 수 있어야 합니다.
- 정보는 많은데 조작은 단순해야 합니다.
- 다크 행정 누아르 분위기를 유지합니다.
- 텍스트 기반 선택형 스토리와 운영 시뮬레이션이 충돌하지 않도록 합니다.
- 자원 변화는 숫자와 게이지를 함께 사용합니다.
- 위험 자원은 상승할수록 불안하게 보이도록 설계합니다.
- 선택지 결과는 즉시 피드백되도록 합니다.

## 주요 UI 컴포넌트

| 컴포넌트 | 용도 |
|---|---|
| Day Header | 현재 일차와 시간대 표시 |
| Resource Bar | 핵심 자원 상태 표시 |
| Alert Badge | 위험/긴급 상태 표시 |
| Report Card | 하루 보고 또는 사건 요약 표시 |
| Choice Button | 선택지 표시 |
| Result Panel | 선택 결과와 수치 변화 표시 |
| Character Card | 인물 상태와 관계 표시 |
| System Log | 최근 사건/결정 이력 표시 |

## GitHub와 Figma 연결 기준

| 항목 | Figma | GitHub |
|---|---|---|
| 화면 구조 | 시각화 | `docs/ui-ux/`에 설명 |
| 화면 목록 | 프레임으로 관리 | `08_UI화면목록` 시트와 문서로 관리 |
| 컴포넌트 | 디자인 컴포넌트 | 컴포넌트 목적 문서화 |
| 변경 이력 | 파일 히스토리 | 작업 로그에 주요 변경 기록 |
| 최종 기준 | 디자인 시안 | 문서와 구현 기준 |

## 새 세션에서 Figma를 사용하는 방식

1. GitHub의 `docs/ui-ux/FIGMA_STRUCTURE.md`를 확인합니다.
2. 실제 Figma 파일명이 `red-ledger-game_적혈의장부_UIUX`인지 확인합니다.
3. 화면을 추가하거나 변경할 때 `08_UI화면목록` 시트와 GitHub 문서를 함께 갱신합니다.
4. 최종 확정된 화면은 GitHub `docs/ui-ux/`에 설명을 남깁니다.

## 현재 상태

- Figma 실제 파일 생성은 아직 확인되지 않았습니다.
- 본 문서는 실제 Figma 파일을 만들기 위한 화면 설계 기준입니다.
