# 첫 화면 흐름 설계

이 문서는 `red-ledger-game`의 첫 실행 화면부터 Day 1 메인 운영 화면까지의 플레이 흐름을 정의합니다.

## 목적

첫 화면 흐름은 플레이어가 게임의 정체성을 이해하고, 복잡한 세계관에 압도되지 않은 상태로 첫 운영 판단에 들어가도록 만드는 구간입니다.

초반에는 모든 세계관을 설명하지 않고 다음 감각만 전달합니다.

```text
이곳은 평범한 공익 기관처럼 보인다.
그러나 기록이 어긋난다.
공식 수요와 이면 수요가 다르다.
누군가 이 모든 것을 감시하고 있다.
```

## 핵심 목표

- 게임 제목과 분위기 각인
- 플레이어 역할 소개
- 혈연센터의 표면과 이면 소개
- 핵심 자원 UI 첫 노출
- 첫 선택지 제공
- Day 1 메인 운영 화면으로 자연스럽게 진입

## 전체 흐름

```text
타이틀 화면
→ 새 게임 / 이어하기 / 설정
→ 프롤로그 문서 화면
→ 부임 통지 화면
→ 봉인 보고서 화면
→ Day 1 시작 보고
→ 핵심 자원 튜토리얼
→ 첫 선택지
→ 선택 결과
→ 메인 운영 화면
```

## SCR-001 타이틀 화면

### 목적

게임의 첫 인상을 전달합니다.

### 화면 요소

- 게임 제목: `적혈의 장부`
- 부제 후보: `The Red Ledger`
- 어두운 배경
- 오래된 장부 또는 봉인 문서 이미지
- 붉은 도장 또는 얇은 강조선
- 메뉴 버튼
  - 새 게임
  - 이어하기
  - 설정

### 톤

과장된 공포보다 차갑고 행정적인 불길함을 우선합니다.

```text
보고서가 조용히 거짓말을 하고 있다는 느낌
```

## SCR-002 프롤로그 문서 화면

### 목적

혈연센터가 정상적인 기관처럼 보이도록 합니다.

### 화면 요소

```text
백십자재단 내부 인사 발령 통지
수신: 서윤재 실장
발령지: 혈연센터 중앙관리실
등급: 일반 문서
```

문서 하단에는 작은 문구가 표시됩니다.

```text
※ 봉인 문서는 야간 결재 이후 열람 가능
```

## SCR-003 부임 통지 화면

### 목적

플레이어의 역할을 명확히 합니다.

### 핵심 문장

```text
당신은 오늘부터 혈연센터의 실무 책임자입니다.
공식 업무는 보관, 일정, 민원, 재고 관리입니다.
비공식 업무는 아직 고지되지 않았습니다.
```

### 버튼

```text
업무 인수 확인
```

## SCR-004 봉인 보고서 화면

### 목적

게임의 이면을 처음 드러냅니다.

### 화면 요소

- 일반 보고서 위에 겹쳐지는 봉인 문서
- 붉은 봉인 표시
- 문서 등급: `가문 열람`
- 발신: `카르테인 가문 혈액관리국`

### 핵심 문장

```text
표면 재고와 실제 재고는 일치하지 않는다.
공식 수요와 가문 수요는 별도로 관리한다.
공급 실패는 외부 노출보다 빠르게 재앙을 부른다.
```

## SCR-005 Day 1 시작 보고

### 목적

Day 시스템과 첫 목표를 소개합니다.

### 화면 요소

```text
Day 1
부임 첫날
상태: 관찰 단계
```

### 오늘의 목표

```text
혈연센터의 표면 업무를 유지하라.
봉인 보고서의 이면 수요를 확인하라.
첫 배분 결정을 내려라.
```

## SCR-006 핵심 자원 튜토리얼

### 목적

자원 UI를 처음 보여줍니다.

### 처음 강조할 자원

```text
핵심 재고
이면 수요
인간 신뢰
언론 노출
가문 만족도
```

나머지 자원은 Day 2~Day 4에서 자연스럽게 해금합니다.

```text
보안 등급
조직 불안
암거래 단서
가문 자금
```

## SCR-007 첫 선택지 화면

### 이벤트

```text
EVT-D01-001 첫 운영 보고
```

### 상황

표면 보고서에는 재고가 안정적이라고 적혀 있습니다. 그러나 봉인 보고서에는 오늘 밤까지 별도 수요를 맞추라는 가문 지시가 붙어 있습니다.

### 선택지

| 선택지 | 톤 | 결과 방향 |
|---|---|---|
| 표면 보고를 기준으로 정석 운영한다 | 정석 | 인간 신뢰 안정, 가문 만족도 하락 |
| 봉인 보고를 우선해 가문 수요를 맞춘다 | 가문 우선 | 가문 만족도 상승, 핵심 재고 감소 |
| 두 보고서의 차이를 조용히 보정한다 | 은폐/중용 | 보안 등급 소모, 단기 안정 |

## SCR-008 선택 결과 화면

### 목적

선택 결과를 짧고 명확하게 보여줍니다.

### 표시 방식

```text
선택 결과 요약
변화한 자원
관련 인물 반응
다음 위험 암시
```

## SCR-009 메인 운영 화면 진입

### 목적

본격적인 하루 운영 화면으로 이동합니다.

### 화면 구조

```text
상단: Day / 시간대 / 상태
중단: 핵심 자원 바
본문: 오늘의 업무 카드
하단: 보고서 / 인물 / 기록 / 종료 버튼
```

### 처음 보이는 업무 카드

```text
오늘의 업무
1. 재고표 확인
2. 가문 수요 확인
3. 민원 기록 확인
4. 야간 보고 준비
```

## 정보 공개 순서

| 순서 | 공개 정보 |
|---|---|
| 1 | 혈연센터는 공익 기관처럼 보인다 |
| 2 | 플레이어는 실무 책임자다 |
| 3 | 봉인 보고서가 존재한다 |
| 4 | 공식 수요와 이면 수요가 다르다 |
| 5 | 공급 부족도 위험하다 |
| 6 | 은폐만으로는 해결되지 않는다 |
| 7 | 중용이 핵심이다 |

## UI 연출 방향

- 문서가 겹쳐지는 방식으로 표면/이면을 표현합니다.
- 처음에는 깔끔한 행정 UI로 시작합니다.
- 봉인 보고서가 열리면서 붉은 강조색이 처음 등장합니다.
- 자원 UI는 보고서의 표처럼 보이게 합니다.
- 공포 연출보다 차가운 위화감을 우선합니다.

- 새 게임 버튼

- 봉인 보고서 화면
- Day 1 시작 보고
- 핵심 자원 5개 첫 노출
- 첫 선택지 3개
- 선택 결과 화면
- 메인 운영 화면 진입
