# 와이어프레임 설계 메모

이 문서는 Figma에서 1차 와이어프레임을 만들 때 사용할 기본 배치와 화면별 구성 요소를 정리합니다.

## 공통 레이아웃 기준

모바일 세로 화면 기준으로 다음 구조를 우선 사용합니다.

```text
┌────────────────────┐
│ Day / 장소 / 상태   │  Header
├────────────────────┤
│ 핵심 자원 요약      │  Resource Bar
├────────────────────┤
│ 주요 보고/사건 카드 │  Main Content
│                    │
├────────────────────┤
│ 선택지 / 조작 버튼  │  Action Area
├────────────────────┤
│ 로그 / 하단 메뉴    │  Footer
└────────────────────┘
```

## SCR-003 메인 운영 화면

### 목적

플레이어가 현재 상황, 핵심 자원, 오늘 처리할 업무를 한눈에 파악하는 화면입니다.

### 구성 요소

- Day Header
- 혈액 재고 게이지
- 가문 자금 표시
- 인간 신뢰 표시
- 언론 노출 위험 표시
- 조직 불안 위험 표시
- 오늘의 주요 업무 카드
- 이벤트 진입 버튼
- 일차 종료 버튼
- 시스템 로그

### 설계 메모

- 가장 중요한 자원은 위쪽에 배치합니다.
- 위험 자원은 색이나 아이콘으로 긴장감을 줍니다.
- 버튼은 2~3개 이하로 단순하게 유지합니다.

## SCR-005 이벤트 발생 화면

### 목적

사건의 분위기와 선택 압박을 전달하는 화면입니다.

### 구성 요소

- 이벤트 제목
- 사건 설명 텍스트
- 관련 인물 카드
- 영향 받을 자원 예고
- 선택지 버튼 2~4개

### 설계 메모

- 사건 설명은 너무 길지 않게 카드형으로 구성합니다.
- 선택지는 텍스트 중심으로 표현합니다.
- 선택지마다 위험성의 뉘앙스를 다르게 둡니다.

## SCR-006 선택 결과 화면

### 목적

플레이어가 선택한 결과와 자원 변화를 즉시 이해하는 화면입니다.

### 구성 요소

- 선택 결과 요약
- 변화한 자원 목록
- 후속 위험 알림
- 관련 인물 반응
- 계속 버튼

### 설계 메모

- 변화량은 `+5`, `-10`처럼 명확히 표시합니다.
- 모든 수치를 한꺼번에 보여주기보다 변화한 항목 중심으로 보여줍니다.
- 결과가 다음 사건으로 이어질 가능성을 암시합니다.

## SCR-002 일차 시작 보고 화면

### 목적

각 Day의 목표와 현재 위기 상황을 요약합니다.

### 구성 요소

- Day 번호
- 오늘의 운영 목표
- 전날 결과 요약
- 주요 위험 예고
- 시작 버튼

## SCR-008 일차 종료 보고 화면

### 목적

하루 동안의 결과와 다음 Day로 넘어가기 전 상태를 정리합니다.

### 구성 요소

- 오늘 처리한 사건 수
- 주요 자원 변화
- 인물/조직 반응
- 가문 평가
- 다음 Day 진입 버튼

## SCR-009 최종 평가 화면

### 목적

7일 운영 결과를 종합해 엔딩 또는 평가를 보여줍니다.

### 구성 요소

- 최종 등급
- 주요 자원 최종값
- 핵심 선택 요약
- 인물별 후일담
- 재시작/기록 보기 버튼

## 컴포넌트 우선 제작 순서

1. Resource Bar
2. Report Card
3. Choice Button
4. Result Panel
5. Character Card
6. Alert Badge
7. System Log

## 연결 기준

- 화면ID는 `SCREEN_LIST.md`와 일치시킵니다.
- Figma 프레임명 앞에는 화면ID를 붙입니다.
- 구현 단계에서는 `src/`의 화면 컴포넌트명과도 연결합니다.
