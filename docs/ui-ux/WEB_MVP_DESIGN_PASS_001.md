## 작업 목적

배포 성공 직후 확인된 Web MVP는 기능 검증용 와이어프레임에 가까웠습니다.

따라서 컨텐츠 확장 전에 다음 문제를 먼저 줄이는 1차 디자인 패스를 진행했습니다.

```text
테스트 페이지 느낌 제거
모바일 하단 버튼 가림 완화
자원 카드 위계 정리
선택지 버튼 질감 강화
최종 보고서 가독성 개선
배경/테두리/그림자/색 대비 정리
```

## 적용 파일

```text
src/web-mvp/styles.css
.github/workflows/web-mvp-pages.yml
```

## 주요 변경

### 1. 모바일 게임형 화면 폭 정리

`app-shell` 최대 폭을 460px로 조정하고, 하단 safe-area 여백을 강화했습니다.

### 2. 배경 질감 개선

단순 어두운 배경에서 벗어나 다음 효과를 추가했습니다.

```text
상단 버건디 라디얼 조명
암부 그라데이션
미세 격자 질감
```

### 3. 카드 컴포넌트 개선

문서 카드, 경고 카드, 일반 카드의 테두리/광택/그림자 위계를 강화했습니다.

### 4. 자원 카드 개선

자원 카드의 숫자 크기, 상태별 색상, 내부 하이라이트를 정리했습니다.

상태 구분:

```text
good
warn
danger
```

### 5. 선택지 버튼 개선

선택지 버튼 높이와 폰트 위계를 키워 모바일 터치 영역과 게임 UI 느낌을 강화했습니다.

### 6. 진행 표시 개선

Day 진행 점의 크기, 간격, 발광 효과를 조정했습니다.

### 7. Actions Pages 실패 반복 차단

GitHub Pages는 최종적으로 branch 기반 배포를 사용합니다.

```text
Source: Deploy from a branch
Branch: main
Folder: / root
```

따라서 기존 Actions 기반 Pages 배포 워크플로우는 자동 실행을 끄고 수동 안내용으로 전환했습니다.
