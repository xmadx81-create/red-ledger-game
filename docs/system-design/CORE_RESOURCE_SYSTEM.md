# 핵심 자원 시스템

이 문서는 `red-ledger-game`의 핵심 자원, 위험 지표, 중용 시스템, 수요/공급 판정 구조를 정의합니다.

## 목적

핵심 자원 시스템은 플레이어가 매일 보고 판단해야 하는 게임의 기본 엔진입니다.

이 게임은 특정 수치를 무조건 높이는 게임이 아닙니다. 핵심은 다음 세 가지를 동시에 관리하는 것입니다.

```text
들키지 않는다.
공급을 끊기게 하지 않는다.
중용을 지킨다.
```

플레이어는 혈액 재고, 가문 수요, 인간 신뢰, 언론 노출, 조직 불안, 암거래 단서 등을 보며 매일 결정을 내려야 합니다.

## 핵심 철학: 최대화가 아니라 중용

일반적인 경영 시뮬레이션은 돈, 생산량, 평판을 높이는 방향으로 설계되는 경우가 많습니다.

하지만 `적혈의 장부`는 다릅니다.

너무 낮아도 위험하고, 너무 높아도 위험한 자원이 존재합니다.

예를 들어 혈액 재고가 부족하면 뱀파이어들이 통제를 잃을 수 있지만, 과도하게 확보하면 기록 불일치, 폐기 문제, 외부 의심이 커질 수 있습니다.

따라서 플레이어의 목표는 자원 최대화가 아니라 `안전 구간 유지`입니다.

## 자원 분류

MVP의 자원은 세 종류로 나눕니다.

| 분류 | 설명 | 예시 |
|---|---|---|
| 핵심 운영 자원 | 매일 직접 관리하는 기본 자원 | 혈액 재고, 혈액 수요, 가문 자금 |
| 외부 위험 지표 | 인간 사회와 외부 노출 관련 위험 | 인간 신뢰, 언론 노출, 보안 등급 |
| 내부/이면 위험 지표 | 가문, 조직, 암거래망 관련 위험 | 가문 만족도, 조직 불안, 암거래 단서 |

## 기본 수치 범위

모든 주요 자원은 MVP에서 기본적으로 `0~100` 범위를 사용합니다.

단, `혈액 재고`, `혈액 수요`, `가문 자금`은 숫자 자원으로 확장 가능성이 있으므로 MVP에서는 0~100 기준으로 단순화합니다.

```text
0 = 최악 또는 고갈
50 = 기본 안정권
100 = 최대 또는 위험 임계치
```

## 자원별 상세 설계

### RES-001 혈액 재고

혈액 재고는 현재 센터가 보유한 사용 가능 혈액량입니다.

- 낮을수록 가문 불만과 폭주 위험이 증가합니다.
- 높을수록 보관, 기록, 폐기, 외부 감사 위험이 증가합니다.
- 혈액 수요와 직접 비교됩니다.

### RES-002 혈액 수요

혈액 수요는 가문과 개별 뱀파이어들이 요구하는 혈액량입니다.

- Day가 진행될수록 변동됩니다.
- 특정 사건에서 급증할 수 있습니다.
- 수요를 반복적으로 충족하지 못하면 폭주 이벤트가 발생합니다.

### RES-003 가문 자금

가문 자금은 은폐, 회유, 보상, 보안 강화, 정보 차단에 사용하는 자원입니다.

- 낮으면 위기에 대응할 수 없습니다.
- 너무 자주 사용하면 장기 운영력이 약화됩니다.
- 일부 선택지는 자금이 부족하면 선택할 수 없습니다.

### RES-004 인간 신뢰

인간 신뢰는 혈연센터가 인간 사회에서 받는 공익적 신뢰입니다.

- 낮으면 민원, 조사, 언론 관심이 증가합니다.
- 너무 높으면 방문자, 후원자, 외부 감사, 언론 인터뷰 요청이 늘어날 수 있습니다.
- 적정 수준의 신뢰가 가장 안정적입니다.

### RES-005 언론 노출

언론 노출은 기자, 제보, 의혹, 보도 위험을 의미합니다.

- 낮을수록 안전합니다.
- 60 이상이면 위험 이벤트가 자주 발생합니다.
- 85 이상이면 폭로 직전 상태입니다.
- 100에 도달하면 은폐 실패 엔딩 또는 긴급 위기 이벤트가 발생합니다.

### RES-006 보안 등급

보안 등급은 기록, 출입, 내부 통제, 정보 차단 체계의 안정성을 의미합니다.

- 낮으면 기록 불일치, 정보 누출, 내부 실수가 증가합니다.
- 일부 은폐 선택지는 보안 등급을 소모합니다.
- 보안 등급이 낮은 상태에서 언론 노출이 높으면 폭로 위험이 급격히 증가합니다.

### RES-007 가문 만족도

가문 만족도는 카르테인 가문이 현재 운영을 어떻게 평가하는지 나타냅니다.

- 낮으면 가문 원로의 강제 개입, 직접 명령, 처벌 이벤트가 발생합니다.
- 높으면 당장은 안정적이지만, 가문이 더 큰 요구를 할 가능성이 생깁니다.
- 적정 수준을 유지하는 것이 중요합니다.

### RES-008 조직 불안

조직 불안은 직원, 협력자, 하부 조직의 동요와 피로를 나타냅니다.

- 높으면 내부 고발, 실수, 기록 누락, 배신 가능성이 증가합니다.
- 조직 불안을 낮추려면 보상, 휴식, 설득, 책임 분산이 필요합니다.
- 조직 불안을 무시하면 Day 6 이후 큰 위기로 이어집니다.

### RES-009 암거래 단서

암거래 단서는 인간 사회 내부의 불법 혈액 거래 흐름을 플레이어가 얼마나 파악했는지를 의미합니다.

이 수치는 단순히 높을수록 좋은 수치가 아닙니다.

- 너무 낮으면 암거래망을 거의 파악하지 못한 상태입니다.
- 적정 수준이면 조용히 감시하고 통제할 수 있습니다.
- 너무 높으면 플레이어 조직이 암거래망과 직접 충돌하거나 외부 노출 위험이 증가합니다.

암거래 요소는 현실의 불법 거래 방법을 설명하는 것이 아니라, 게임 내 위험 지표와 사건 단서로만 다룹니다.

## 일일 수요/공급 판정

매일 시작 시 다음 흐름으로 수요/공급을 판정합니다.

```text
1. 오늘의 혈액 수요 계산
2. 사용 가능한 혈액 재고 확인
3. 공식 공급량과 부족분 계산
4. 부족분이 있으면 가문 만족도 하락
5. 부족분이 반복되면 폭주 위험 증가
6. 과잉 재고가 있으면 보안/기록 위험 증가
```

## 수요/공급 기본 공식

MVP에서는 복잡한 수식보다 명확한 판정식을 사용합니다.

```text
공급 차이 = 혈액 재고 - 혈액 수요
```

판정 기준:

| 공급 차이 | 판정 | 결과 방향 |
|---:|---|---|
| -30 이하 | 심각한 부족 | 폭주 위험 증가, 가문 만족도 크게 하락 |
| -29~-10 | 부족 | 가문 만족도 하락, 추가 사건 가능 |
| -9~20 | 적정 | 안정권 |
| 21~40 | 과잉 | 기록 불일치, 보안 부담 증가 |
| 41 이상 | 위험한 과잉 | 언론 노출, 폐기 문제, 내부 의심 증가 |

## 중용 구간

각 자원은 안전 구간을 가집니다.

안전 구간 안에 있으면 안정 보너스 또는 위기 감소 효과를 받을 수 있습니다.

안전 구간을 벗어나면 즉시 실패하지는 않지만, 관련 사건 발생 확률이 높아집니다.

```text
낮은 위험 구간 → 안정
경고 구간 → 사건 발생 가능성 증가
위험 구간 → 강제 위기 이벤트 가능
최대/최소 도달 → 실패 또는 엔딩 분기
```

## 위험 단계

| 단계 | 조건 | 의미 |
|---|---|---|
| 안정 | 안전 구간 내부 | 일반 운영 가능 |
| 경고 | 안전 구간 이탈 | 관련 경고 메시지 표시 |
| 위험 | 임계치 접근 | 사건 발생 확률 증가 |
| 붕괴 | 0 또는 100 도달 | 실패/강제 이벤트/엔딩 분기 |

## 자원 상호작용

자원은 독립적으로 움직이지 않습니다.

| 변화 | 연쇄 효과 |
|---|---|
| 혈액 재고 부족 | 가문 만족도 하락, 조직 불안 상승 |
| 혈액 재고 과잉 | 보안 등급 하락, 언론 노출 상승 가능 |
| 혈액 수요 증가 | 공급 부족 위험 증가 |
| 인간 신뢰 하락 | 언론 노출 상승, 외부 조사 가능 |
| 인간 신뢰 과잉 상승 | 외부 접촉 증가, 보안 부담 증가 |
| 언론 노출 상승 | 보안 등급 소모, 은폐 선택지 비용 증가 |
| 조직 불안 상승 | 내부 실수, 암거래 연루, 기록 누락 증가 |
| 가문 만족도 하락 | 강제 지시, 수요 증가, 처벌 이벤트 가능 |
| 암거래 단서 상승 | 통제 가능성 증가, 직접 충돌 위험도 증가 |

## 선택지 결과 표기 방식

플레이어에게 모든 결과를 숫자로 완전히 공개하지 않습니다.

MVP에서는 다음 방식으로 표시합니다.

| 표시 | 의미 |
|---|---|
| `혈액 재고 소폭 증가` | +1~+5 |
| `혈액 재고 증가` | +6~+12 |
| `혈액 재고 크게 증가` | +13 이상 |
| `언론 노출 소폭 증가` | +1~+5 |
| `조직 불안 증가` | +6~+12 |
| `보안 등급 크게 하락` | -13 이상 |

개발/밸런싱용 내부 데이터에는 정확한 숫자를 저장하고, 플레이어 UI에는 요약 표현을 우선 사용합니다.

## 밸런싱 원칙

- Day 1은 학습 구간으로 큰 실패가 발생하지 않게 합니다.
- Day 2부터 공급 부족 압박을 줍니다.
- Day 3부터 언론 노출을 본격적으로 사용합니다.
- Day 4부터 조직 불안을 올립니다.
- Day 5에서 가문 수요가 급증합니다.
- Day 6에서 암거래 단서와 복합 위기가 연결됩니다.
- Day 7은 누적 결과를 평가합니다.
