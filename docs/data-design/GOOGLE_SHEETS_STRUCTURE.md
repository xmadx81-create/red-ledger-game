# Google Sheets 관리표 구조

이 문서는 `red-ledger-game` 프로젝트의 Google Sheets 관리표 구조를 정의합니다.

## 목적

Google Sheets는 프로젝트의 표 형식 데이터를 관리하는 도구입니다. GitHub 문서가 기준과 기록을 담당한다면, Google Sheets는 작업 목록, 게임 데이터, 밸런싱 수치, MVP 진행표처럼 행과 열로 관리해야 하는 자료를 담당합니다.

## 권장 스프레드시트명

```text
red-ledger-game_관리표
```

또는 한글 병기형:

```text
red-ledger-game_적혈의장부_관리표
```

## 권장 탭 구조

```text
00_대시보드
01_작업목록
02_MVP_7일구조
03_자원
04_캐릭터
05_이벤트
06_선택지
07_밸런싱
08_UI화면목록
99_변경로그
```

## 탭별 역할

| 탭 | 역할 |
|---|---|
| `00_대시보드` | 전체 진행률, 우선순위, 현재 작업 요약 |
| `01_작업목록` | 해야 할 일, 담당 도구, 상태, 마감일 관리 |
| `02_MVP_7일구조` | 1차 MVP의 Day 1~Day 7 운영 흐름 관리 |
| `03_자원` | 혈액 재고, 자금, 위신, 신뢰 등 핵심 자원 정의 |
| `04_캐릭터` | 주요 인물, 역할, 성향, 등장 조건 관리 |
| `05_이벤트` | 일차별/조건별 이벤트 목록 관리 |
| `06_선택지` | 이벤트별 선택지와 결과값 관리 |
| `07_밸런싱` | 수치 변화량, 난이도, 자원 증감 테스트 |
| `08_UI화면목록` | 필요한 화면, Figma 연결, 구현 상태 관리 |
| `99_변경로그` | 시트 구조 변경 이력 관리 |

## GitHub와 Sheets의 연결 기준

| 항목 | Google Sheets | GitHub |
|---|---|---|
| 작업 목록 | 행 단위 관리 | 요약/결정사항 기록 |
| 자원 수치 | 수치표 관리 | 시스템 설계 문서화 |
| 캐릭터 목록 | 항목 관리 | 인물 설정 문서화 |
| 이벤트 목록 | 데이터 관리 | 이벤트 구조 문서화 |
| 선택지 결과 | 수치 변화 관리 | 규칙과 의도 설명 |
| UI 화면 목록 | 상태 관리 | 화면 설계 기준 문서화 |

## 새 세션에서 Sheets를 사용하는 방식

1. GitHub의 `docs/data-design/GOOGLE_SHEETS_STRUCTURE.md`를 먼저 확인합니다.
2. 실제 Google Sheets 문서가 생성되어 있으면 해당 시트의 탭 구조와 헤더를 비교합니다.
3. 차이가 있으면 `99_변경로그`에 변경 이유를 기록합니다.
4. 확정된 구조 변경은 GitHub 문서에도 반영합니다.

## 현재 상태

- Google Sheets 실제 파일 생성은 아직 진행하지 않았습니다.
- 본 문서는 실제 스프레드시트를 만들기 위한 기준 설계 문서입니다.
