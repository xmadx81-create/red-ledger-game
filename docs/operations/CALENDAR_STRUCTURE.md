# Google Calendar 일정 운영 기준

이 문서는 `red-ledger-game` 프로젝트의 Google Calendar 운영 기준을 정의합니다.

## 목적

Google Calendar는 프로젝트 일정, 마감일, 점검일, 회의 일정을 관리하는 도구입니다.

GitHub가 기준 문서와 작업 이력을 관리하고, Google Sheets가 작업 목록과 상태를 관리한다면, Google Calendar는 실제 시간 기반 리마인드와 일정 관리를 담당합니다.

## 권장 캘린더명

```text
red-ledger-game_적혈의장부_일정
```

## 일정 분류

| 분류 | 용도 | 예시 |
|---|---|---|
| 전처리 | 도구 연결, 구조 확정, 문서 정리 | GitHub 구조 정리, Sheets 구조 설계 |
| 기획 | 세계관, 시스템, 캐릭터, 이벤트 설계 | MVP 7일 구조 확정 |
| 데이터 | Google Sheets, CSV, 밸런싱 데이터 작업 | 자원표 1차 작성 |
| UIUX | Figma, 와이어프레임, 화면 흐름 작업 | 메인 운영 화면 설계 |
| 개발 | 코드 구현, 기능 추가, 테스트 | Day 진행 루프 구현 |
| 검토 | 리뷰, 수정, 보완, 회의 | 1차 MVP 검토 |
| 마감 | 특정 산출물 완료 목표 | MVP 기획서 v1 완료 |

## 일정 제목 규칙

일정 제목은 다음 형식을 권장합니다.

```text
[분류] 작업명
```

예시:

```text
[전처리] Google Drive 폴더 생성 확인
[기획] MVP 7일 운영 구조 확정
[데이터] 자원표 1차 작성
[UIUX] 메인 운영 화면 와이어프레임
[검토] 1차 MVP 구조 리뷰
```


## 기본 반복 일정 제안

전처리 단계에서는 다음 반복 점검을 권장합니다.

| 반복 일정 | 주기 | 목적 |
|---|---|---|
| 프로젝트 상태 점검 | 주 1회 | GitHub 체크리스트와 작업 로그 확인 |
| 데이터 구조 점검 | 주 1회 | Sheets 구조와 GitHub 문서 불일치 확인 |
| UI/UX 점검 | 주 1회 | Figma 화면ID와 문서/시트 연결 확인 |
| 작업 로그 정리 | 주 1회 | 새 세션 인수인계 가능 상태 유지 |

## 1차 전처리 일정 초안

| 순서 | 일정명 | 완료 기준 |
|---|---|---|
| 1 | GitHub 연결 및 구조 정리 | 저장소 접근, 폴더 구조, 작업 규칙 완료 |
| 2 | Google Drive 구조 확정 | Drive 폴더 구조 기준 문서 완료 |
| 3 | Google Sheets 구조 확정 | 탭 구조, 헤더, CSV 템플릿 완료 |
| 4 | Figma 설계 기준 확정 | 화면ID, 페이지 구조, 와이어프레임 기준 완료 |
| 5 | Calendar 일정 기준 확정 | 일정 분류, 제목 규칙, 반복 점검 기준 완료 |
| 6 | Gmail 알림 기준 확정 | 라벨, 알림, 공유 메일 기준 완료 |

## GitHub와 Calendar 연결 기준

| 항목 | Google Calendar | GitHub |
|---|---|---|
| 일정 제목 | 실행해야 할 작업 표시 | 작업 기준 문서 링크 |
| 일정 설명 | 완료 조건과 링크 기록 | 상세 문서와 작업 로그 |
| 일정 완료 | 실제 시간 관리 | `CHECKLIST.md` 완료 처리 |
| 회의 일정 | 캘린더 이벤트 | 회의록은 `docs/operations/`에 저장 |
| 마감일 | 리마인드 | 관련 이슈/작업 문서와 연결 |

## Google Sheets와 Calendar 연결 기준

- Google Sheets의 `01_작업목록`에 있는 `마감일`은 Calendar 일정과 맞춥니다.
- Calendar 일정명과 Sheets 작업명은 가능하면 동일하게 유지합니다.
- 일정이 바뀌면 Sheets의 마감일과 상태도 함께 변경합니다.
- 중요한 일정 변경은 `99_변경로그` 또는 GitHub 작업 로그에 남깁니다.

## 새 세션에서 Calendar를 사용하는 방식

1. GitHub의 `docs/operations/CALENDAR_STRUCTURE.md`를 확인합니다.
2. 실제 캘린더 이름이 `red-ledger-game_적혈의장부_일정`인지 확인합니다.
3. Google Sheets `01_작업목록`의 마감일과 Calendar 일정을 비교합니다.
4. 완료된 일정은 `CHECKLIST.md` 또는 관련 작업표에 반영합니다.
5. 중요한 일정 변경은 `WORK_LOG.md` 또는 `99_변경로그`에 남깁니다.

## 현재 상태

- Google Calendar 실제 캘린더 생성은 아직 확인되지 않았습니다.
- 본 문서는 실제 캘린더를 만들고 일정을 등록하기 위한 기준 문서입니다.
